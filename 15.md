# AWS SAP-C02: Storage Solution for Archived Documents

## Understanding the Problem

The company needs a **low-cost storage solution** for a **large number of archived documents** that will be accessed by employees via a **corporate intranet (client VPN attached to a VPC)**.

Key requirements and constraints:

| Requirement/Constraint | Implication |
| :--- | :--- |
| **Store archived documents** | Requires a durable, cost-effective storage class. |
| **Lowest cost** | Crucial, pointing to highly optimized or infrequent access storage. |
| **Accessible via corporate intranet (Client VPN $\rightarrow$ VPC)** | Access must be private, likely using VPC endpoints or private networking. |
| **Data must not be accessible to the public** | Requires private network access and strong access control (e.g., S3 bucket policy or security group). |
| **Copies of data held on physical media elsewhere** | Suggests data is non-critical to operations and can tolerate reduced redundancy. |
| **Low number of requests** | Infrequent access storage classes are ideal. |
| **Availability and speed of retrieval are not concerns** | Confirms suitability for archive/infrequent access tiers, including those with higher retrieval latency. |

The most critical factors are **Lowest Cost** and **Archived/Infrequent Access** with **Private Access**.

## Services Explanation

| Service | Description | Relevance to Question |
| :--- | :--- | :--- |
| **Amazon S3 (Simple Storage Service)** | Object storage offering high durability, scalability, and various storage classes. | Good for large archives; offers lowest-cost storage classes. |
| **S3 One Zone-Infrequent Access (S3 One Zone-IA)** | Storage class for infrequently accessed data, stored in a **single Availability Zone (AZ)**. It's $20\%$ cheaper than S3 Standard-IA, but data is lost in the event of AZ destruction. | Meets **low-cost** and **infrequent access** needs. Acceptable risk given the data is a "copy" (non-critical). |
| **S3 Glacier Deep Archive** | The lowest-cost S3 storage class for long-term data archival, accessed once or twice a year. Retrieval time is typically 12 hours. | Meets **lowest cost** and **archive** needs, but retrieval time might be too long even if "speed of retrieval is not a concern." |
| **S3 Interface Endpoint (PrivateLink)** | Provides a private connection from the VPC to S3, keeping all traffic within the AWS network. | Meets the **private access** requirement. |
| **Amazon EC2** | Provides resizable compute capacity in the cloud. | Involves management overhead and higher running cost than pure object storage. |
| **Amazon EFS (Elastic File System)** | Scalable, elastic NFS file system. | File storage is typically more expensive than object storage for simple archives. Requires managing an EC2 instance to serve. |
| **EFS One Zone-Infrequent Access (EFS One Zone-IA)** | EFS storage class for infrequently accessed data stored in a single AZ. | Similar risk profile to S3 One Zone-IA, but EFS is generally more expensive than S3 per GB. Requires an EC2 instance. |
| **Amazon EBS (Elastic Block Store) Cold HDD (sc1)** | Lowest-cost magnetic EBS volume type, designed for throughput-intensive workloads (like big data) with infrequent access. | Requires managing an EC2 instance and EBS volume. Cost-effective for block storage, but still requires the cost of an EC2 instance. |
| **S3 Website Hosting** | A configuration of an S3 bucket to serve static content over HTTP/HTTPS. | Not necessary and adds complexity/potential public exposure if not strictly locked down.

## Correct Option(s) Explanation

The correct answer is **A**.

**A. Create an Amazon S3 bucket. Configure the S3 bucket to use the S3 One Zone-Infrequent Access (S3 One Zone-IA) storage class as default. Create an S3 interface endpoint. Configure the S3 bucket to allow access only through that endpoint.**

1.  **Lowest Cost:** **S3 One Zone-IA** is significantly cheaper than S3 Standard, S3 Standard-IA, EFS, and EBS for infrequent access data. This addresses the primary requirement for the **LOWEST cost**. The risk of data loss in a single AZ is acceptable because the data are "copies" held elsewhere (AWS S3 documentation indicates S3 One Zone-IA is $20\%$ lower cost than S3 Standard-IA, per *AWS S3 Storage Classes* documentation).
2.  **Archival/Infrequent Access:** This storage class is explicitly designed for **infrequently accessed** data with a lower cost per GB. The low number of requests is also a perfect fit.
3.  **Private Access:** An **S3 interface endpoint** ensures that the data is accessed **privately** from the VPC (where the Client VPN connects), satisfying the security requirement ("Data must not be accessible to the public").
4.  **Simplicity/No Compute Overhead:** S3 is a managed object storage service and does not require an EC2 instance, avoiding the recurring cost and management overhead of EC2, EFS, or EBS.
5.  **Note on S3 Website Hosting:** The option also includes "Configure the S3 bucket for website hosting." While this is *unnecessary* for a private intranet-only solution and might *increase risk* if misconfigured, the *combination* of **S3 One Zone-IA** and **S3 interface endpoint** provides the absolute lowest-cost, private, archival solution, making it the best answer despite the extra website hosting step. *For the exam, focus on the core cost and access mechanisms.*

## Incorrect Options Explanation

### D. Create an Amazon S3 bucket. Configure the S3 bucket to use the S3 Glacier Deep Archive storage class as default. Configure the S3 bucket for website hosting. Create an S3 interface endpoint. Configure the S3 bucket to allow access only through that endpoint.

* **Primary Flaw: Retrieval Time/Latency.** While **S3 Glacier Deep Archive** is the absolute *lowest cost per GB*, its standard retrieval time is typically 12 hours (up to 48 hours for bulk retrieval) (per *AWS S3 Storage Classes* documentation). Even though "speed of retrieval is not a concern," this level of latency is typically unacceptable for employees accessing documents through an "intranet." **S3 One Zone-IA** offers millisecond access. Therefore, S3 One Zone-IA is the most practical *lowest-cost* solution for active *intranet* access.

### B. Launch an Amazon EC2 instance that runs a web server. Attach an Amazon Elastic File System (Amazon EFS) file system to store the archived data in the EFS One Zone-Infrequent Access (EFS One Zone-IA) storage class Configure the instance security groups to allow access only from private networks.

* **Primary Flaw: Cost and Complexity.** This solution requires **running and managing an EC2 instance** and **an EFS file system**. EFS has a higher base cost per GB than S3, even with the One Zone-IA class. The combination of EC2 hourly charges, EC2 management, and higher EFS storage costs makes this significantly **more expensive** than the serverless S3 solution in Option A.

### C. Launch an Amazon EC2 instance that runs a web server Attach an Amazon Elastic Block Store (Amazon EBS) volume to store the archived data. Use the Cold HDD (sc1) volume type. Configure the instance security groups to allow access only from private networks.

* **Primary Flaw: Cost, Scalability, and Management.** This solution requires **running and managing an EC2 instance** and **an EBS volume**. The EC2 instance adds a fixed, recurring cost, making it more expensive than S3. EBS volumes are also *limited in size* (up to $16 \text{TiB}$ for $\text{sc}1$), are *less durable* than S3 ($99.8\%$ for $\text{sc}1$ vs. $99.999999999\%$ for S3), and are **not scalable** like S3 object storage (you must provision the size upfront). For a large and growing number of documents, S3 is the superior and lower-cost solution.
