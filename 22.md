A company that has multiple AWS accounts is using AWS Organizations. The company’s AWS accounts host VPCs, Amazon EC2 instances, and containers.  
The company’s compliance team has deployed a security tool in each VPC where the company has deployments. The security tools run on EC2 instances and send information to the AWS account that is dedicated for the compliance team. The company has tagged all the compliance-related resources with a key of “costCenter” and a value or “compliance”.  
The company wants to identify the cost of the security tools that are running on the EC2 instances so that the company can charge the compliance team’s AWS account. The cost calculation must be as accurate as possible.  
What should a solutions architect do to meet these requirements?

- A. In the management account of the organization, activate the costCenter user-defined tag. Configure monthly AWS Cost and Usage Reports to save to an Amazon S3 bucket in the management account. Use the tag breakdown in the report to obtain the total cost for the costCenter tagged resources.
- B. In the member accounts of the organization, activate the costCenter user-defined tag. Configure monthly AWS Cost and Usage Reports to save to an Amazon S3 bucket in the management account. Schedule a monthly AWS Lambda function to retrieve the reports and calculate the total cost for the costCenter tagged resources.
- C. In the member accounts of the organization activate the costCenter user-defined tag. From the management account, schedule a monthly AWS Cost and Usage Report. Use the tag breakdown in the report to calculate the total cost for the costCenter tagged resources.
- D. Create a custom report in the organization view in AWS Trusted Advisor. Configure the report to generate a monthly billing summary for the costCenter tagged resources in the compliance team’s AWS account.
# AWS SAP-C02 Cost Allocation Question Analysis

## Understanding the Problem

The company is using **AWS Organizations** with multiple accounts and needs to implement an accurate **cost allocation** mechanism. Specifically, they need to identify the exact cost of **security tools** running on **Amazon EC2 instances** across various **member accounts**. These security tools are tagged with the user-defined tag **`costCenter:compliance`**. The goal is to charge this cost back to the **compliance team's dedicated AWS account**. The key requirements are:
1.  Identify the cost of resources (EC2 instances) tagged with **`costCenter:compliance`**.
2.  The cost calculation must be as **accurate as possible**.
3.  The solution must work across **multiple AWS accounts** within an **AWS Organization**.

## Correct Option(s) Explanation

The correct option is **A**.

**A. In the management account of the organization, activate the costCenter user-defined tag. Configure monthly AWS Cost and Usage Reports to save to an Amazon S3 bucket in the management account. Use the tag breakdown in the report to obtain the total cost for the costCenter tagged resources.**

* **Activate the Tag:** **User-defined Cost Allocation Tags** (like `costCenter`) *must* be activated in the **AWS Billing console** of the **management account** for them to appear in the CUR for the entire organization. This step is mandatory. (Reference: AWS documentation on "Activating user-defined cost allocation tags").
* **Use AWS Cost and Usage Report (CUR):** The requirement is for the cost calculation to be **as accurate as possible**. The **CUR** is the **most granular** and comprehensive cost and usage dataset provided by AWS. It includes line-item details, resource IDs, and all activated cost allocation tags, making it the **authoritative source** for accurate chargeback/showback. (Reference: AWS documentation on "What is the AWS Cost and Usage Report?").
* **Report Configuration:** CUR is configured *once* and provides data for all linked accounts (member accounts) under the organization when created in the management account.
* **Obtain Cost:** The CUR, saved to S3, is a comprehensive file that can be queried (e.g., using Amazon Athena or other tools) to filter specifically on the `costCenter:compliance` tag key-value pair to derive the total cost.

## Incorrect Options Explanation

| Option | Reason for Incorrectness |
| :--- | :--- |
| **B. In the member accounts of the organization, activate the costCenter user-defined tag. Configure monthly AWS Cost and Usage Reports to save to an Amazon S3 bucket in the management account. Schedule a monthly AWS Lambda function to retrieve the reports and calculate the total cost for the costCenter tagged resources.** | **Incorrect Tag Activation Location.** User-defined **Cost Allocation Tags** for **AWS Organizations** are managed and activated **centrally** in the **management account's** Billing console, *not* individually in the member accounts. If activated only in member accounts, they won't reliably appear in the organization-level CUR. |
| **C. In the member accounts of the organization activate the costCenter user-defined tag. From the management account, schedule a monthly AWS Cost and Usage Report. Use the tag breakdown in the report to calculate the total cost for the costCenter tagged resources.** | **Incorrect Tag Activation Location and Missing Granularity Tool.** Similar to B, tag activation must be in the **management account**. Additionally, while the AWS Cost Explorer provides some tag breakdown, the prompt asks for the **most accurate** calculation. Directly querying the **CUR** (as suggested in A) offers superior granularity and detail compared to relying solely on the Cost Explorer's summary views. |
| **D. Create a custom report in the organization view in AWS Trusted Advisor. Configure the report to generate a monthly billing summary for the costCenter tagged resources in the compliance team’s AWS account.** | **Incorrect Service for Granular Cost Allocation.** **AWS Trusted Advisor** is primarily a best-practices and optimization tool. It **does not** provide the detailed, tag-based, and hourly cost data required for accurate, auditable **chargeback/showback**. The **AWS Cost and Usage Report (CUR)** is the authoritative and required source for this level of detail. |
