A company creates an AWS Control Tower landing zone to manage and govern a multi-account AWS environment. The company's security team will deploy preventive controls and detective controls to monitor AWS services across all the accounts. The security team needs a centralized view of the security state of all the accounts.

Which solution will meet these requirements?

A. From the AWS Control Tower management account, use AWS CloudFormation StackSets to deploy an AWS Config conformance pack to all accounts in the organization.
B. Enable Amazon Detective for the organization in AWS Organizations. Designate one AWS account as the delegated administrator for Detective.
C. From the AWS Control Tower management account, deploy an AWS CloudFormation stack set that uses the automatic deployment option to enable Amazon Detective for the organization.
D. Enable AWS Security Hub for the organization in AWS Organizations. Designate one AWS account as the delegated administrator for Security Hub.



# AWS SAP-C02 Question Analysis — Q335

## 📋 Analysis

### 1. Understanding the Problem

The environment is a typical enterprise multi-account architecture managed by AWS Control Tower. The company wants:

- Preventive controls
- Detective controls
- Centralized security visibility across all AWS accounts

This means the company needs a service that can:

- Aggregate security findings organization-wide
- Integrate with AWS Organizations
- Support centralized governance
- Provide a unified dashboard for the security team

The key phrase is:
> "centralized view of the security state of all the accounts"

This directly aligns with AWS Security Hub capabilities.

- --

### 2. Options Analysis

#### ✅ Option D (CORRECT): Enable AWS Security Hub organization-wide with a delegated administrator

This is the correct architecture.

AWS Security Hub is designed specifically for:

- Centralized security posture management
- Multi-account aggregation
- Organization-wide security findings
- Compliance reporting
- Cross-service security visibility

Why this works well:

1. Native AWS Organizations Integration
   - Security Hub integrates directly with AWS Organizations.
   - A delegated administrator account can centrally manage findings from all member accounts.
   - Newly added accounts can be automatically enrolled.

2. Centralized Security Dashboard
   - Security Hub aggregates findings from:
     - AWS Config
     - GuardDuty
     - Inspector
     - IAM Access Analyzer
     - Macie
     - Partner security tools
   - The security team gets a single operational dashboard.

3. Works with Control Tower Governance
   - Control Tower provides governance guardrails.
   - Security Hub provides centralized operational visibility and compliance posture.

4. Enterprise Security Operations Best Practice
   Typical enterprise landing zone pattern:
   - Management account → governance only
   - Security account → delegated admin for security services
   - Log archive account → centralized logging

This exactly matches AWS recommended multi-account security architecture.

Official references:
- AWS Security Hub and Organizations:
  https://docs.aws.amazon.com/securityhub/latest/userguide/securityhub-accounts-orgs.html
- AWS Control Tower:
  https://docs.aws.amazon.com/controltower/latest/userguide/what-is-control-tower.html

- --

#### ❌ Option A (INCORRECT): Deploy AWS Config conformance packs with StackSets

This solution only addresses compliance evaluation partially.

Why it is insufficient:

1. AWS Config Is Not a Central Security Posture Platform
   - Config evaluates resource compliance.
   - Conformance packs apply collections of Config rules.
   - However, Config does not provide comprehensive security aggregation.

2. Missing Unified Security Findings
   - No centralized security operations dashboard comparable to Security Hub.
   - No cross-service finding aggregation.

3. Operational Complexity
   - StackSets deployment introduces unnecessary management overhead.
   - Security Hub organization integration is simpler and purpose-built.

Important SAP-C02 distinction:

- AWS Config → configuration compliance engine
- AWS Security Hub → centralized security posture management

Conformance packs are commonly integrated INTO Security Hub, not used as a replacement.

Official reference:
- AWS Config conformance packs:
  https://docs.aws.amazon.com/config/latest/developerguide/conformance-packs.html

- --

#### ❌ Option B (INCORRECT): Enable Amazon Detective organization-wide

Amazon Detective solves a different problem.

Why this fails:

1. Detective Focuses on Investigation
   - Detective helps analyze:
     - suspicious activity
     - API behavior
     - threat relationships
     - attack timelines

2. Not Designed for Central Governance Visibility
   - It is not a compliance dashboard.
   - It does not provide organization-wide security posture management.

3. Wrong Security Operations Layer
   Think operationally:

   - Security Hub → identify and aggregate findings
   - Detective → investigate findings deeply

Detective is commonly used AFTER alerts are generated.

Official reference:
- Amazon Detective:
  https://docs.aws.amazon.com/detective/latest/userguide/what-is-detective.html

- --

#### ❌ Option C (INCORRECT): Use StackSets to deploy Amazon Detective organization-wide

This option combines:
- wrong service choice
- unnecessary deployment complexity

Why it is incorrect:

1. Detective Still Does Not Meet Requirements
   - No centralized security posture dashboard
   - No governance-focused aggregation

2. StackSets Are Unnecessary Here
   - AWS Organizations delegated admin integration already exists.
   - Native organization onboarding is preferred.

3. Common SAP-C02 Distractor
   AWS exams often include:
   - technically feasible deployment methods
   - but architecturally incorrect services

Always prioritize:
> correct managed service first, deployment mechanism second.

### 3. Hands-On Knowledge

Real enterprise Control Tower deployments usually implement this structure:

```text
AWS Organizations
│
├── Management Account
│   └── Governance only
│
├── Security OU
│   ├── Security Hub Delegated Admin
│   ├── GuardDuty Admin
│   ├── IAM Access Analyzer
│   └── Detective Admin
│
├── Log Archive OU
│   └── Centralized CloudTrail / Config logs
│
└── Workload Accounts
```

Typical production implementation steps:

1. Deploy Control Tower landing zone
2. Create dedicated security account
3. Enable Security Hub organization integration
4. Configure delegated administrator
5. Auto-enable member accounts
6. Aggregate findings centrally
7. Integrate with SIEM systems

CLI example:

```bash
aws securityhub enable-organization-admin-account \
  - -admin-account-id 123456789012
```

Enable automatic onboarding:

```bash
aws securityhub update-organization-configuration \
  - -auto-enable
```

Important operational insight:

Security Hub itself is mainly an aggregation layer.

The actual findings often come from:
- GuardDuty → threat detection
- Inspector → vulnerabilities
- Config → compliance
- Macie → sensitive data discovery
- IAM Access Analyzer → unintended access exposure

Cost consideration in enterprises:

- Security Hub pricing scales per finding/check.
- Large organizations should tune:
  - enabled standards
  - aggregation regions
  - integrated services
  - retention strategy

Common SAP-C02 exam trap:

Many candidates confuse:
- Config aggregation
vs
- Security Hub centralized posture management

Remember:

```text
AWS Config = "Are resources compliant?"
Security Hub = "What is the overall security posture?"
```

* *面试官可能追问 (Possible Follow-up Questions)**：

1. "What is the difference between AWS Security Hub, GuardDuty, and Detective?"
   → 回答要点 (Key Answer Points)：GuardDuty performs intelligent threat detection using logs and ML. Security Hub aggregates and centralizes security findings and compliance posture. Detective helps investigate and analyze suspicious activity after findings are generated.

2. "Why use delegated administrator accounts in AWS Organizations?"
   → 回答要点 (Key Answer Points)：Delegated administrators reduce operational risk on the management account, improve separation of duties, and align with enterprise least-privilege governance practices. AWS recommends running security operations from dedicated security accounts.
