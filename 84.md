A company is planning to migrate its business-critical applications from an on-premises data center to AWS. The company has an on-premises installation of a  
Microsoft SQL Server Always On cluster. The company wants to migrate to an AWS managed database service. A solutions architect must design a heterogeneous database migration on AWS.  
Which solution will meet these requirements?  

- A. Migrate the SQL Server databases to Amazon RDS for MySQL by using backup and restore utilities.
- B. Use an AWS Snowball Edge Storage Optimized device to transfer data to Amazon S3. Set up Amazon RDS for MySQL. Use S3 integration with SQL Server features, such as BULK INSERT.
- C. Use the AWS Schema Conversion Tool to translate the database schema to Amazon RDS for MeSQL. Then use AWS Database Migration Service (AWS DMS) to migrate the data from on-premises databases to Amazon RDS.
- D. Use AWS DataSync to migrate data over the network between on-premises storage and Amazon S3. Set up Amazon RDS for MySQL. Use S3 integration with SQL Server features, such as BULK INSERT.

# AWS SAP-C02 Exam Question Analysis: Heterogeneous Database Migration

### **Analysis**

#### **1. Understanding the Problem**
The scenario describes a **heterogeneous database migration**. This is the key phrase in the requirement.
* **Source:** On-premises Microsoft SQL Server (SQL Server engine).
* **Destination:** Amazon RDS for MySQL (MySQL engine).
* **Constraint:** The engines are different (Heterogeneous).

In a homogeneous migration (e.g., SQL Server to SQL Server), you can often use native tools like backup and restore or log shipping. However, in a heterogeneous migration, the data types, stored procedures, and database schemas are not compatible between the source and the destination. Therefore, a two-step process is required:
1.  **Convert the Schema:** Translate SQL Server definitions to MySQL definitions.
2.  **Migrate the Data:** Move the actual rows of data into the new schema.

#### **2. Services Explanation**

* **AWS Schema Conversion Tool (SCT):**
    * A desktop application used specifically for heterogeneous migrations.
    * It automatically converts the source database schema and a majority of the database code objects (views, stored procedures, functions) to a format compatible with the target database.
    * It provides an assessment report highlighting items that cannot be converted automatically and require manual intervention.

* **AWS Database Migration Service (AWS DMS):**
    * A managed service that migrates databases to AWS quickly and securely.
    * It supports both homogeneous (e.g., Oracle to Oracle) and heterogeneous (e.g., Oracle to Aurora) migrations.
    * DMS handles the data movement (Full Load) and can keep the source and target in sync (Change Data Capture - CDC) to minimize downtime.



#### **3. Correct Option Explanation**
**Option C is correct.**
This option aligns perfectly with the standard AWS architecture for heterogeneous migrations.
1.  **Schema Conversion:** You cannot simply load data if the tables don't exist or if the data types don't match. **AWS SCT** is the required tool to translate the MS SQL Server schema to the MySQL format.
2.  **Data Migration:** Once the empty schema is applied to the target RDS instance, **AWS DMS** connects to the source and target to move the data.

This is the documented best practice for moving from a commercial engine (SQL Server) to an open-source engine (MySQL/Aurora).

#### **4. Incorrect Options Explanation**

* **A. Migrate the SQL Server databases to Amazon RDS for MySQL by using backup and restore utilities.**
    * **Incorrect.** Native backup files (`.bak`) from Microsoft SQL Server are binary files specific to the SQL Server engine. MySQL has no mechanism to read, understand, or restore a Microsoft SQL Server backup file. This method only works for Homogeneous migrations (SQL Server to SQL Server).

* **B. Use an AWS Snowball Edge... Use S3 integration with SQL Server features, such as BULK INSERT.**
    * **Incorrect.** While Snowball moves data, the restoration method described is invalid. `BULK INSERT` is a T-SQL command used by SQL Server. You cannot use SQL Server commands to insert data into an RDS for MySQL instance. Furthermore, RDS for MySQL does not support S3 integration for restoring SQL Server data formats.

* **D. Use AWS DataSync... Use S3 integration with SQL Server features, such as BULK INSERT.**
    * **Incorrect.** Similar to Option B, this fails on the restoration mechanism. DataSync is an excellent tool for moving files (NAS/SAN data) to S3, but it does not solve the database engine incompatibility. The target database (MySQL) cannot consume the raw SQL Server data files or backups using SQL Server commands.
