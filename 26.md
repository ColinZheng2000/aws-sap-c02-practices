A company has created an OU in AWS Organizations for each of its engineering teams. Each OU owns multiple AWS accounts. The organization has hundreds of  
AWS accounts.  
A solutions architect must design a solution so that each OU can view a breakdown of usage costs across its AWS accounts.  
Which solution meets these requirements?  

- A. Create an AWS Cost and Usage Report (CUR) for each OU by using AWS Resource Access Manager. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.
- B. Create an AWS Cost and Usage Report (CUR) from the AWS Organizations management account. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.
- C. Create an AWS Cost and Usage Report (CUR) in each AWS Organizations member account. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.
- D. Create an AWS Cost and Usage Report (CUR) by using AWS Systems Manager. Allow each team to visualize the CUR through Systems Manager OpsCenter dashboards.

## AWS SAP Exam Question Analysis: Viewing Usage Costs Across OUs

### Analysis

#### Understanding the Problem

The company uses **AWS Organizations** to manage **hundreds of AWS accounts** structured into **Organizational Units (OUs)**, where each OU corresponds to an engineering team and contains multiple accounts. The requirement is to design a solution that allows **each OU** (i.e., each team) to **view a breakdown of usage costs across its AWS accounts**. This necessitates a centralized cost reporting mechanism that can aggregate and then allow for a granular, team-specific view of costs.

---

#### Services Explanation

| Service | Description | Relevance to Solution |
| :--- | :--- | :--- |
| **AWS Organizations** | An account management service that allows you to consolidate multiple AWS accounts into an organization that you create and centrally manage. It uses a root, OUs, and member accounts. | **Crucial:** The cost data needs to be aggregated from all accounts in the Organization. The **management account** is the central point for billing and CUR generation for the entire Organization. |
| **AWS Cost and Usage Report (CUR)** | The most comprehensive source of information about your AWS costs and usage. It lists all usage for every service and account, which can be aggregated and stored in an Amazon S3 bucket. | **Crucial:** CUR is the *only* official AWS mechanism that provides the detailed, account-level cost data required for this breakdown across the entire Organization. |
| **Amazon QuickSight** | A scalable, serverless, machine learning-powered business intelligence (BI) service that can connect to and visualize data from various sources, including CUR data in S3. | **Crucial:** Provides the required visualization dashboard and access control to show the relevant cost breakdown to each OU/team. |
| **AWS Resource Access Manager (RAM)** | Used to share AWS resources securely across AWS accounts or within your AWS Organization. | **Not Applicable:** CURs are a data product stored in S3, not a resource that needs to be shared via RAM like a Subnet or Transit Gateway. |
| **AWS Systems Manager (SSM)** | Provides a unified interface for operational data and automates operational tasks across your AWS resources. OpsCenter is for operational work items (OpsItems). | **Not Applicable:** SSM is for operations, not cost reporting and visualization. It cannot aggregate CUR data or replace QuickSight for BI dashboards. |

---

#### Correct Option(s) Explanation

**B. Create an AWS Cost and Usage Report (CUR) from the AWS Organizations management account. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.**

1.  **Centralized CUR Generation:** The **AWS Organizations management account** is the *payer account* and the **only account** that can generate a **single CUR** that includes cost and usage data for *all member accounts* in the Organization. This meets the requirement for viewing costs *across* all AWS accounts. (Refer to the official AWS documentation on CURs and Organizations).
2.  **Visualization:** **Amazon QuickSight** is the appropriate **Business Intelligence (BI) tool** for visualizing the massive amount of data contained within the CUR (which is stored in an S3 bucket).
3.  **Meeting OU Requirement:** By using QuickSight, the Solutions Architect can create dashboards with **Row-Level Security (RLS)** or by filtering the data to display costs only for the accounts **belonging to a specific OU**. This allows each team (OU) to view their specific cost breakdown while leveraging a single, central CUR.

---

#### Incorrect Options Explanation

**A. Create an AWS Cost and Usage Report (CUR) for each OU by using AWS Resource Access Manager. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.**
* **Incorrect Mechanism:** CURs are generated centrally from the **management account** for the entire organization. You cannot generate a CUR *per OU* directly, nor is **AWS Resource Access Manager (RAM)** the correct tool for sharing or managing access to the CUR data itself (which is typically accessed via an S3 bucket policy).

**C. Create an AWS Cost and Usage Report (CUR) in each AWS Organizations member account. Allow each team to visualize the CUR through an Amazon QuickSight dashboard.**
* **Incorrect Scope:** A CUR created in a **member account** can *only* report on the usage for that specific individual account. To view a breakdown of costs *across its AWS accounts* (i.e., across all accounts in the OU and the whole Organization), you **must** use the CUR generated by the **management account**. Creating hundreds of individual CURs is inefficient and doesn't solve the aggregation problem.

**D. Create an AWS Cost and Usage Report (CUR) by using AWS Systems Manager. Allow each team to visualize the CUR through Systems Manager OpsCenter dashboards.**
* **Incorrect Service Usage:** **AWS Systems Manager** and its **OpsCenter** are used for operational management, incident response, and handling operational data (OpsItems). They are **not** the correct tools for creating, aggregating, or visualizing the detailed, financial data from a CUR. **Amazon QuickSight** is the appropriate visualization tool. CURs are generated via the Billing and Cost Management console/API, not Systems Manager.
