A company's solutions architect needs to provide secure Remote Desktop connectivity to users for Amazon EC2 Windows instances that are hosted in a VPC. The solution must integrate centralized user management with the company's on-premises Active Directory. Connectivity to the VPC is through the internet. The company has hardware that can be used to establish an AWS Site-to-Site VPN connection.

Which solution will meet these requirements MOST cost-effectively?

A. Deploy a managed Active Directory by using AWS Directory Service for Microsoft Active Directory. Establish a trust with the on-premises Active Directory. Deploy an EC2 instance as a bastion host in the VPC. Ensure that the EC2 instance is joined to the domain. Use the bastion host to access the target instances through RDP.
B. Configure AWS Single Sign-On to integrate with the on-premises Active Directory by using the AWS Directory Service for Microsoft Active Directory AD Connector. Configure permission sets against user groups for access to AWS Systems Manager. Use Systems Manager Fleet Manager to access the target instances through RDP.
C. Implement a VPN between the on-premises environment and the target VPEnsure that the target instances are joined to the on-premises Active Directory domain over the VPN connection. Configure RDP access through the VPN. Connect from the company's network to the target instances.
D. Deploy a managed Active Directory by using AWS Directory Service for Microsoft Active Directory. Establish a trust with the on-premises Active Directory. Deploy a Remote Desktop Gateway on AWS by using an AWS Quick Start. Ensure that the Remote Desktop Gateway is joined to the domain. Use the Remote Desktop Gateway to access the target instances through RDP.


Suggest Answer: B

---

## 📋 Analysis

### Understanding the Problem

A solutions architect needs to provide secure RDP access to EC2 Windows instances in a VPC. The critical constraints are: (1) centralized user management must integrate with the existing on-premises Active Directory, (2) connectivity to the VPC is via the internet, (3) the company has VPN-capable hardware available, and (4) the solution must be the **most cost-effective**. This means minimizing recurring infrastructure costs while maintaining security and usability.

### ✅ Correct Option: B

**AWS IAM Identity Center (formerly AWS SSO) + AD Connector + Systems Manager Fleet Manager for browser-based RDP**

This is the most cost-effective solution because:

1. **No Bastion Host / RD Gateway Required**: Systems Manager Fleet Manager provides browser-based RDP access directly to EC2 Windows instances through the AWS Management Console. This eliminates the need for any bastion host, jump server, or Remote Desktop Gateway — saving the cost of running additional EC2 instances 24/7.

2. **AD Connector Minimizes Directory Costs**: AD Connector is a proxy that forwards authentication requests to the on-premises Active Directory. It requires only two small EC2 instances (unlike AWS Managed Microsoft AD which runs full domain controllers). AD Connector pricing is significantly lower than a full Managed AD deployment.

3. **AWS IAM Identity Center is Free**: There is no additional charge for IAM Identity Center itself. You only pay for the underlying AD Connector directory.

4. **Centralized User Management**: AD Connector integrates seamlessly with on-premises AD — user accounts, group memberships, and credentials remain in the on-premises directory. IAM Identity Center maps AD groups to permission sets, controlling who can access Systems Manager Fleet Manager.

5. **Security & Audit**: All Fleet Manager actions are logged in AWS CloudTrail. IAM policies provide fine-grained access control (e.g., which instances a user can access). No need to expose RDP ports (3389) to the internet.

### ❌ Incorrect Options

**Option A — Managed AD + Bastion Host**

- AWS Managed Microsoft AD runs two domain controllers across AZs — this is more expensive than AD Connector's lightweight proxy instances.
- A bastion host is an additional EC2 instance that must be patched, monitored, and secured. It represents a recurring compute cost and an additional attack surface.
- The bastion host must also be domain-joined, adding management complexity.

**Option C — VPN + Join Instances to On-Premises AD Directly**

- AWS Site-to-Site VPN incurs hourly per-tunnel charges (~$0.05/hour per VPN connection) plus data transfer costs. While not enormous, this adds up over time.
- Relying on VPN connectivity for AD authentication means that any VPN interruption breaks RDP access — reducing operational resilience.
- RDP traffic traversing the VPN tunnel back to the corporate network and then to EC2 introduces latency and consumes VPN bandwidth.
- This approach offers no centralized access management or auditing beyond what the AD domain provides natively.

**Option D — Managed AD + Remote Desktop Gateway Quick Start**

- The RD Gateway Quick Start deploys a full Remote Desktop Gateway farm (typically multiple EC2 instances behind a load balancer). This is the most infrastructure-heavy option with **significant ongoing compute costs**.
- AWS Managed AD is more expensive than AD Connector.
- RD Gateway requires Windows Server licensing (included in EC2 pricing but still reflected in instance costs).
- While architecturally similar to traditional on-premises RDP solutions, this approach carries the highest operational and financial overhead.

### 🔧 Hands-On Knowledge

- **Systems Manager Fleet Manager**: A feature of AWS Systems Manager that provides a browser-based interactive access experience for Windows (RDP) and Linux (SSH) instances. It tunnels traffic through the SSM Agent — **no inbound security group rules needed** for port 3389 or 22.
- **AD Connector vs. Managed AD Decision Matrix**:
  | Feature | AD Connector | AWS Managed Microsoft AD |
  |---|---|---|
  | Cost | Lower (proxy only) | Higher (full DCs) |
  | On-premises AD dependency | Required | Optional (trust) |
  | Supports MFA | Yes (via RADIUS) | Yes |
  | Use case | Hybrid, AD on-premises | Cloud-native or trust-based hybrid |
- **SSM Agent Requirements**: EC2 instances must have the SSM Agent installed, an IAM instance profile with `AmazonSSMManagedInstanceCore`, and connectivity to the SSM service endpoints (via VPC endpoints in private subnets, or NAT gateway in public subnets).
- **Cost Optimization**: For environments with many Windows instances, this architecture avoids Windows Server licensing costs for bastion hosts and RD Gateway servers.
