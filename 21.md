A security engineer determined that an existing application retrieves credentials to an Amazon RDS for MySQL database from an encrypted file in Amazon S3. For the next version of the application, the security engineer wants to implement the following application design changes to improve security:  
The database must use strong, randomly generated passwords stored in a secure AWS managed service.  
The application resources must be deployed through AWS CloudFormation.  
The application must rotate credentials for the database every 90 days.  
A solutions architect will generate a CloudFormation template to deploy the application.  
Which resources specified in the CloudFormation template will meet the security engineer’s requirements with the LEAST amount of operational overhead?

- A. Generate the database password as a secret resource using AWS Secrets Manager. Create an AWS Lambda function resource to rotate the database password. Specify a Secrets Manager RotationSchedule resource to rotate the database password every 90 days.
- B. Generate the database password as a SecureString parameter type using AWS Systems Manager Parameter Store. Create an AWS Lambda function resource to rotate the database password. Specify a Parameter Store RotationSchedule resource to rotate the database password every 90 days.
- C. Generate the database password as a secret resource using AWS Secrets Manager. Create an AWS Lambda function resource to rotate the database password. Create an Amazon EventBridge scheduled rule resource to trigger the Lambda function password rotation every 90 days.
- D. Generate the database password as a SecureString parameter type using AWS Systems Manager Parameter Store. Specify an AWS AppSync DataSource resource to automatically rotate the database password every 90 days.

# AWS Certified Solutions Architect - Professional (SAP-C02) Exam Question Analysis

## Understanding the Problem

The security engineer needs to implement **strong, randomly generated, and securely stored database passwords** for an **Amazon RDS for MySQL** instance. The entire solution must be deployed via **AWS CloudFormation**, and the password must be automatically **rotated every 90 days**. The key constraint is meeting these requirements with the **LEAST amount of operational overhead**.

| Requirement | Service / Feature Focus |
| :--- | :--- |
| **Secure Storage & Random Generation** | AWS Secrets Manager or AWS Systems Manager Parameter Store (SecureString) |
| **Deployment** | AWS CloudFormation |
| **Automatic 90-day Rotation** | Built-in rotation mechanism vs. custom-built (Lambda/EventBridge) |
| **Constraint** | LEAST operational overhead |

***

## Services Explanation

| Service | Description | Relevance to Problem |
| :--- | :--- | :--- |
| **AWS Secrets Manager** | A service to help protect secrets needed to access applications, services, and IT resources. It enables you to easily rotate, manage, and retrieve database credentials, API keys, and other secrets throughout their lifecycle. **It includes built-in rotation for supported databases (like RDS).** | Highly relevant for secure storage, generation, and built-in automatic rotation with low overhead. |
| **AWS Systems Manager Parameter Store** | Provides secure, hierarchical storage for configuration data management and secrets. Secrets are stored as **SecureString** parameters. **It does NOT have a built-in rotation feature.** | Relevant for secure storage, but lacks the automatic rotation feature required. |
| **AWS CloudFormation** | An infrastructure-as-code service that allows you to model and provision AWS and third-party resources. **Secrets Manager secrets and rotation schedules can be defined as resources in a template.** | Essential for deploying the entire application stack and resources. |
| **AWS Lambda** | A serverless compute service that runs code in response to events. Often used to create custom logic, such as a custom password rotation mechanism. | Necessary only if a built-in rotation mechanism isn't available or sufficient, which increases operational overhead. |
| **Amazon EventBridge** | A serverless event bus that connects application components. Can be used for scheduled events (cron-like) to trigger a Lambda function. | Can be used to trigger custom rotation, but is an extra resource compared to built-in rotation. |
| **AWS AppSync** | A managed service that allows applications to interact with data using GraphQL. **It is not designed to manage or rotate database credentials.** | Irrelevant for this specific security requirement. |

***

## Correct Option(s) Explanation

### Correct Answer: A

**A. Generate the database password as a secret resource using AWS Secrets Manager. Create an AWS Lambda function resource to rotate the database password. Specify a Secrets Manager RotationSchedule resource to rotate the database password every 90 days.**

* **Secret Generation and Storage:** Using **AWS Secrets Manager** meets the requirement for strong, randomly generated passwords stored in a secure, AWS-managed service. The secret resource can be configured in CloudFormation to automatically generate credentials for an RDS database.
* **Rotation:** **AWS Secrets Manager** provides **native, built-in rotation** for Amazon RDS (including MySQL). When defined in CloudFormation, the `AWS::SecretsManager::RotationSchedule` resource automatically manages the rotation process. While Secrets Manager uses a **Lambda function** behind the scenes to perform the actual rotation logic, **the entire rotation setup—including the necessary Lambda function—is provided and managed by Secrets Manager as part of its feature set** (or deployed by CloudFormation automatically when configuring the rotation schedule for an RDS database).
* **Operational Overhead:** This solution uses the *managed rotation feature* of Secrets Manager, which is the definition of **LEAST operational overhead**. It requires configuring just two resources in CloudFormation (the Secret and the Rotation Schedule) to achieve all requirements, relying on AWS to manage the complexity of the rotation logic and scheduling.

***

## Incorrect Options Explanation

### B. Generate the database password as a SecureString parameter type using AWS Systems Manager Parameter Store. Create an AWS Lambda function resource to rotate the database password. Specify a Parameter Store RotationSchedule resource to rotate the database password every 90 days.

* **SecureString:** Parameter Store's `SecureString` is suitable for storage, but Parameter Store **does not offer a native rotation feature** like Secrets Manager.
* **Parameter Store RotationSchedule:** This resource **does not exist** in AWS CloudFormation or the Parameter Store service. This option is factually incorrect and would necessitate a custom-built solution, which increases operational overhead.

### C. Generate the database password as a secret resource using AWS Secrets Manager. Create an AWS Lambda function resource to rotate the database password. Create an Amazon EventBridge scheduled rule resource to trigger the Lambda function password rotation every 90 days.

* **Secrets Manager:** Correctly identifies Secrets Manager for storage.
* **Rotation:** This approach proposes building a **custom rotation mechanism** using an AWS Lambda function and an Amazon EventBridge schedule. While technically functional (this is what Secrets Manager *does* internally), it is **not the solution with the LEAST operational overhead**. The engineer would have to write, test, and maintain the Lambda code and manage the EventBridge schedule, whereas Option A utilizes the fully managed, built-in rotation feature of Secrets Manager.

### D. Generate the database password as a SecureString parameter type using AWS Systems Manager Parameter Store. Specify an AWS AppSync DataSource resource to automatically rotate the database password every 90 days.

* **SecureString:** Parameter Store does not have a native rotation feature.
* **AppSync:** AWS AppSync is a GraphQL service and is **not designed** to handle automated credential rotation for an RDS database. This option introduces an irrelevant service and is factually incorrect regarding rotation capabilities.
