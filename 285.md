A company maintains information on premises in approximately 1 million.csv files that are hosted on a VM. The data initially is 10 TB in size and grows at a rate of 1 TB each week. The company needs to automate backups of the data to the AWS Cloud.

Backups of the data must occur daily. The company needs a solution that applies custom filters to back up only a subset of the data that is located in designated source directories. The company has set up an AWS Direct Connect connection.

Which solution will meet the backup requirements with the LEAST operational overhead?

A. Use the Amazon S3 CopyObject API operation with multipart upload to copy the existing data to Amazon S3. Use the CopyObject API operation to replicate new data to Amazon S3 daily.
B. Create a backup plan in AWS Backup to back up the data to Amazon S3. Schedule the backup plan to run daily.
C. Install the AWS DataSync agent as a VM that runs on the on-premises hypervisor. Configure a DataSync task to replicate the data to Amazon S3 daily.
D. Use an AWS Snowball Edge device for the initial backup. Use AWS DataSync for incremental backups to Amazon S3 daily.


# SAP-C02 Analysis — AWS DataSync for Large-Scale On-Premises Backup

## Understanding the Problem

The scenario includes several important architectural requirements:

- On-premises VM hosting:
  - ~1 million `.csv` files
  - 10 TB initial dataset
  - growth 1 TB weekly
- Daily automated backups required
- Need custom filters:
  - only specific directories/subsets
- AWS Direct Connect already exists
- Requirement:
  - least operational overhead

This is fundamentally:
- large-scale hybrid data transfer
- ongoing incremental synchronization
- filtered backup workloads

The ideal AWS-native service is:
# AWS DataSync

# Incorrect Options

---

## A. Use S3 CopyObject API and multipart upload

### Why Incorrect

This is operationally heavy and technically flawed.

Problems:

- `CopyObject` copies objects:
  - already inside S3
  - not on-prem files
- Would require:
  - custom scripts
  - upload orchestration
  - retry logic
  - scheduling
  - integrity management
- Managing 1 million files manually:
  - very complex

Multipart upload:
- helps large object uploads
- does NOT solve automation requirements

The exam tests recognition that:
- custom scripting is not preferred when managed services exist

---

## B. Use AWS Backup to back up to S3

### Why Incorrect

AWS Backup does NOT:
- directly back up arbitrary on-prem file systems into S3

AWS Backup supports:
- EBS
- EFS
- DynamoDB
- FSx
- RDS
- VMware workloads (via specific integration)

But:
- not generic file-directory filtering to S3 in this way

Also:
- AWS Backup is not optimized for:
  - millions of files
  - selective directory synchronization

This is a common distractor.

---

## D. Snowball Edge + DataSync

### Why Incorrect

This is partially valid technically but not optimal.

Snowball is typically used when:
- network bandwidth is insufficient
- no high-speed connectivity exists
- initial transfer would take too long

But the company already has:
- AWS Direct Connect

Therefore:
- Snowball introduces unnecessary logistics and operational overhead

Key SAP exam principle:
- avoid physical transfer devices if network transfer is sufficient

Also:
- 10 TB is relatively modest for Direct Connect + DataSync
