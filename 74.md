A company is running an application in the AWS Cloud. The application collects and stores a large amount of unstructured data in an Amazon S3 bucket. The S3 bucket contains several terabytes of data and uses the S3 Standard storage class. The data increases in size by several gigabytes every day.  
  
The company needs to query and analyze the data. The company does not access data that is more than 1 year old. However, the company must retain all the data indefinitely for compliance reasons.  
  
Which solution will meet these requirements MOST cost-effectively?

- A. Use S3 Select to query the data. Create an S3 Lifecycle policy to transition data that is more than 1 year old to S3 Glacier Deep Archive.
- B. Use Amazon Redshift Spectrum to query the data. Create an S3 Lifecycle policy to transition data that is more than 1 year old 10 S3 Glacier Deep Archive.
- C. Use an AWS Glue Data Catalog and Amazon Athena to query the data. Create an S3 Lifecycle policy to transition data that is more than 1 year old to S3 Glacier Deep Archive.
- D. Use Amazon Redshift Spectrum to query the data. Create an S3 Lifecycle policy to transition data that is more than 1 year old to S3 Intelligent-Tiering.

## Analysis

### Understanding the Problem

The problem asks for the **most cost-effective** solution to meet two primary requirements:
1.  **Query & Analysis:** The company needs to query and analyze several terabytes of unstructured data stored in Amazon S3.
2.  **Data Lifecycle & Retention:** Data older than 1 year is not accessed, but *all* data must be retained indefinitely for compliance.

The solution must be the cheapest one that satisfies both points. This means we need to find the cheapest query method for S3 data and the cheapest storage method for long-term, unaccessed compliance data.

---

### Services Explanation

* **Amazon S3 (Simple Storage Service):** A scalable object storage service. It offers different **storage classes** optimized for various access patterns and costs.
    * **S3 Standard:** Used here. Designed for frequently accessed data. It's the most expensive storage class (of the ones mentioned).
    * **S3 Intelligent-Tiering:** Automatically moves data between a frequent and an infrequent access tier based on usage patterns. It's ideal for data with **unknown or changing access patterns**. It incurs a small per-object monitoring fee.
    * **S3 Glacier Deep Archive:** The lowest-cost storage class in AWS. It is designed for long-term data archival and retention (e.g., compliance) where data is accessed extremely rarely. Retrieval times are long (typically 12+ hours).

* **S3 Lifecycle Policy:** An automated mechanism to manage your objects' entire lifecycle. You can define rules to automatically transition objects to other (cheaper) S3 storage classes or delete them after a certain period.

* **Amazon Athena:** A serverless, interactive query service that makes it easy to analyze data *directly in Amazon S3* using standard SQL. You pay **per query** (based on the amount of data scanned), with no infrastructure to manage.

* **AWS Glue Data Catalog:** A fully managed, persistent metadata repository. For serverless services like Athena and Redshift Spectrum, the Data Catalog holds the schema and location of your S3 data, allowing the query engine to understand the data's structure.

* **Amazon Redshift Spectrum:** A feature of Amazon Redshift that allows you to run SQL queries against data directly in your S3 data lake. It requires a running **Amazon Redshift cluster**, which has a persistent cost (even if the cluster itself is paused, Spectrum requires an active cluster to run queries).

* **S3 Select:** A feature that allows you to use simple SQL expressions to filter the contents of a *single* S3 object and retrieve only the subset of data you need. It is not a full-scale analytics engine for querying across many objects.

---

### Correct Option Explanation (C)

This option is the most cost-effective because it correctly pairs the most economical solution for each requirement.

1.  **Query (Athena + Glue):** For querying data *in place* on S3, Amazon Athena is almost always the most cost-effective choice for ad-hoc analysis.
    * **Serverless:** You do not pay for a provisioned cluster (like you do with Redshift).
    * **Pay-per-query:** You are only billed for the data your queries scan. If you don't run queries, you don't pay (except for negligible Glue Catalog storage).
    * This perfectly fits the "query and analyze" requirement without the cost overhead of a 24/7 data warehouse. The Glue Data Catalog is a necessary component for Athena to map the unstructured data to a queryable schema.

2.  **Lifecycle (S3 Glacier Deep Archive):**
    * **Known Access Pattern:** The problem explicitly states data older than 1 year is *not accessed*.
    * **Compliance Requirement:** The data must be retained *indefinitely*.
    * **S3 Glacier Deep Archive** is purpose-built for this exact use case. It provides the **absolute lowest storage cost** in AWS, making it the perfect choice for long-term compliance data that you do not plan to access. An S3 Lifecycle policy is the correct tool to automate this transition after 365 days.

As noted in the official AWS documentation, "S3 Glacier Deep Archive is Amazon S3's lowest-cost storage class... for data that is retained for 7-10 years or longer." (Source: [Amazon S3 Storage Classes](https://aws.amazon.com/s3/storage-classes/))

---

### Incorrect Options Explanation

* **A:** The storage half is correct (S3 Glacier Deep Archive). However, **S3 Select** is the wrong query tool. S3 Select is for filtering data *within* a single object, not for running complex analytical queries *across* potentially millions of objects. It cannot fulfill the "query and analyze" requirement at this scale.

* **B:** The storage half is correct (S3 Glacier Deep Archive). However, **Amazon Redshift Spectrum** is not the *most* cost-effective query solution. While powerful, it requires a running Amazon Redshift cluster. This cluster incurs a constant hourly cost, making it significantly more expensive than Athena's serverless pay-per-query model for this use case.

* **D:** This option is incorrect on both parts.
    * **Query:** As with option B, **Redshift Spectrum** is more expensive than Athena.
    * **Lifecycle:** **S3 Intelligent-Tiering** is the wrong choice. This class is for *unknown* or *unpredictable* access patterns. We have a *very* predictable pattern (accessed for 1 year, then never). Intelligent-Tiering also incurs a monitoring fee and its archive tiers are not as cheap as S3 Glacier Deep Archive. Using it here would be significantly more expensive than a direct lifecycle policy to Deep Archive.
