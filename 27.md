A company is storing data on premises on a Windows file server. The company produces 5 GB of new data daily.  
The company migrated part of its Windows-based workload to AWS and needs the data to be available on a file system in the cloud. The company already has established an AWS Direct Connect connection between the on-premises network and AWS.  
Which data migration strategy should the company use?  

- A. Use the file gateway option in AWS Storage Gateway to replace the existing Windows file server, and point the existing file share to the new file gateway
- B. Use AWS DataSync to schedule a daily task to replicate data between the on-premises Windows file server and Amazon FSx
- C. Use AWS Data Pipeline to schedule a daily task to replicate data between the on-premises Windows file server and Amazon Elastic File System (Amazon EFS)
- D. Use AWS DataSync to schedule a daily task to replicate data between the on-premises Windows file server and Amazon Elastic File System (Amazon EFS)

### Understanding the Problem

The company needs to continuously replicate **5 GB of daily new data** from an **on-premises Windows file server (SMB source)** to a **cloud file system** to support existing **Windows-based workloads migrated to AWS**. The cloud file system **must be mountable and accessible by Windows EC2 instances**.

### Correct Option(s) Explanation

**B. Use AWS DataSync to schedule a daily task to replicate data between the on-premises Windows file server and Amazon FSx**

- **AWS DataSync** is the correct service for high-performance, automated, and scheduled data transfer from an **SMB source** (the on-premises Windows file server) over the **Direct Connect** link.
    
- **Amazon FSx for Windows File Server** is the correct and **natively supported** destination file system for a **Windows-based workload**. It provides a fully managed, highly available file system that speaks the **SMB protocol**, which the Windows EC2 instances require to access the data in the cloud.
    
- This combination satisfies all requirements: fast replication (DataSync), support for the source protocol (SMB), and a compatible cloud file system destination for the Windows workload (FSx for Windows File Server/SMB).
    

> **Reference:** AWS documentation on FSx for Windows File Server states it "provides fully managed, highly reliable, and scalable file storage that is accessible via the industry-standard **Server Message Block (SMB) protocol**."

---

### Incorrect Options Explanation

**A. Use the file gateway option in AWS Storage Gateway to replace the existing Windows file server, and point the existing file share to the new file gateway**

- **Incorrect Goal:** File Gateway is a hybrid service for providing _cached on-premises access_ to S3, not the primary tool for _migrating and replicating_ data to a scalable, fully-managed file system for _cloud-based EC2 instances_ to consume. DataSync is the dedicated replication tool.
    

**C. Use AWS Data Pipeline to schedule a daily task to replicate data between the on-premises Windows file server and Amazon Elastic File System (Amazon EFS)**

- **Incorrect Service & Target:** **Data Pipeline** is not the correct service for high-performance file replication. **EFS** is not supported by Windows EC2 instances.
    

**D. Use AWS DataSync to schedule a daily task to replicate data between the on-premises Windows file server and Amazon Elastic File System (Amazon EFS)**

- **Incorrect Target:** While **DataSync** is the correct transfer tool, **Amazon EFS** is **not supported** for mounting by the **Windows-based workload** in AWS. The Windows EC2 instances would be unable to access the replicated file system. This makes the entire strategy non-viable for the problem posed.
