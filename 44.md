A company runs a new application as a static website in Amazon S3. The company has deployed the application to a production AWS account and uses Amazon CloudFront to deliver the website. The website calls an Amazon API Gateway REST API. An AWS Lambda function backs each API method.  
  
The company wants to create a CSV report every 2 weeks to show each API Lambda function’s recommended configured memory, recommended cost, and the price difference between current configurations and the recommendations. The company will store the reports in an S3 bucket.  
  
Which solution will meet these requirements with the LEAST development time?

- A. Create a Lambda function that extracts metrics data for each API Lambda function from Amazon CloudWatch Logs for the 2-week period. Collate the data into tabular format. Store the data as a .csv file in an S3 bucket. Create an Amazon EventBridge rule to schedule the Lambda function to run every 2 weeks.
- B. Opt in to AWS Compute Optimizer. Create a Lambda function that calls the ExportLambdaFunctionRecommendations operation. Export the .csv file to an S3 bucket. Create an Amazon EventBridge rule to schedule the Lambda function to run every 2 weeks.
- C. Opt in to AWS Compute Optimizer. Set up enhanced infrastructure metrics. Within the Compute Optimizer console, schedule a job to export the Lambda recommendations to a .csv file. Store the file in an S3 bucket every 2 weeks.
- D. Purchase the AWS Business Support plan for the production account. Opt in to AWS Compute Optimizer for AWS Trusted Advisor checks. In the Trusted Advisor console, schedule a job to export the cost optimization checks to a .csv file. Store the file in an S3 bucket every 2 weeks.

## Analysis

### Understanding the Problem

The core requirement is to generate a bi-weekly CSV report that provides **recommendations** for Lambda function memory configurations, the **estimated cost** of those recommendations, and the **price difference** compared to the current setup. The report must be stored in an S3 bucket. A critical constraint is to achieve this with the **least development time**.

This means we should look for a managed AWS service that can:

1. Analyze Lambda performance.
    
2. Generate memory and cost-saving recommendations.
    
3. Export this specific data in CSV format.
    
4. Be automated on a schedule.
    

### Services Explanation

- **AWS Compute Optimizer:** This is a free, opt-in service that uses machine learning to analyze the configuration and utilization metrics of your AWS resources. It generates recommendations to help you improve performance and optimize costs. For AWS Lambda, it specifically analyzes functions and recommends an optimal memory configuration. It provides "finding" data (e.g., "Over-provisioned," "Under-provisioned") and the cost impact of its recommendations.
    
- **AWS Lambda:** A serverless compute service. Its pricing is based on the number of requests and the execution duration (in milliseconds), multiplied by the amount of memory configured. This is why optimizing memory is a key cost-saving lever.
    
- **Amazon EventBridge:** A serverless event bus that can also act as a powerful scheduler (similar to `cron`). It's the standard way to trigger AWS services (like Lambda) on a recurring schedule (e.g., "every 2 weeks").
    
- **Amazon S3:** Scalable object storage, perfect for storing generated reports like a .csv file.
    
- **Amazon CloudWatch:** The monitoring and observability service that collects metrics. Compute Optimizer _uses_ CloudWatch metrics (like `Duration` and `MaxMemoryUsed`) as the input for its analysis.
    
- **AWS Trusted Advisor:** A service that provides real-time guidance to help you provision your resources following AWS best practices. It has checks for cost optimization, but it's a higher-level guidance tool and _consumes_ recommendations from services like Compute Optimizer. It is not the primary tool for _generating_ or _exporting_ this specific, detailed data.
    

### Correct Option(s) Explanation

**Option B:** This is the correct solution as it perfectly aligns with the requirements using the least amount of effort.

1. **Opt in to AWS Compute Optimizer:** This activates the service that performs the complex analysis and generates the _exact_ data required (recommended memory, costs, and price difference). This saves all the development time of building a custom analysis engine.
    
2. **Use `ExportLambdaFunctionRecommendations`:** This is a specific API operation provided by Compute Optimizer. As noted in the [AWS Compute Optimizer documentation](https://docs.aws.amazon.com/compute-optimizer/latest/APIReference/API_ExportLambdaFunctionRecommendations.html), this single API call does _exactly_ what's needed: it exports the Lambda recommendations directly to a specified S3 bucket in a .csv file format.
    
3. **Create a Lambda function:** The Lambda function's code is minimal. It only needs to contain the SDK call to `ExportLambdaFunctionRecommendations`, passing in the S3 bucket destination. This is extremely low development effort.
    
4. **Create an Amazon EventBridge rule:** This is the standard, simple way to schedule the Lambda function to run every 2 weeks.
    

This solution directly uses the right managed service (Compute Optimizer) and its specific API for automated exports (`ExportLambdaFunctionRecommendations`), minimizing custom development.

### Incorrect Option(s) Explanation

- **Option A:** This solution is incorrect because it violates the "LEAST development time" constraint. It requires you to build a complex, custom solution from scratch. You would need to:
    
    - Write code to fetch metrics from CloudWatch Logs (which is less efficient than using CloudWatch Metrics).
        
    - Develop your own ML model or sophisticated algorithm to analyze those metrics and _calculate_ memory recommendations.
        
    - Build a cost-modeling engine to estimate current vs. recommended costs and the price difference.
        
        This is a significant engineering effort, replicating a service (Compute Optimizer) that AWS provides for free.
        
- **Option C:** This solution is incorrect because it describes a feature that does not exist. While you can _manually_ export recommendations from the Compute Optimizer console as a one-time action, there is **no built-in feature in the console to _schedule_ a recurring export job**. The only way to automate this export is by using the API, as described in Option B.
    
- **Option D:** This solution is incorrect because it confuses the roles of multiple services.
    
    - AWS Compute Optimizer does not require a Business Support plan; it's a free service available to all accounts after opting in.
        
    - Trusted Advisor _can_ display _some_ recommendations from Compute Optimizer, but it is not the service used to generate or export the detailed recommendation data.
        
    - You cannot schedule a recurring export of Compute Optimizer data from the Trusted Advisor console. This option proposes an incorrect workflow using the wrong tools.
