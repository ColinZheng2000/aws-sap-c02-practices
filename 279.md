A scientific company needs to process text and image data from an Amazon S3 bucket. The data is collected from several radar stations during a live, time-critical phase of a deep space mission. The radar stations upload the data to the source S3 bucket. The data is prefixed by radar station identification number.
The company created a destination S3 bucket in a second account. Data must be copied from the source S3 bucket to the destination S3 bucket to meet a compliance objective. The replication occurs through the use of an S3 replication rule to cover all objects in the source S3 bucket.
One specific radar station is identified as having the most accurate data. Data replication at this radar station must be monitored for completion within 30 minutes after the radar station uploads the objects to the source S3 bucket.
What should a solutions architect do to meet these requirements?

A. Set up an AWS DataSync agent to replicate the prefixed data from the source S3 bucket to the destination S3 bucket. Select to use all available bandwidth on the task, and monitor the task to ensure that it is in the TRANSFERRING status. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to trigger an alert if this status changes.
B. In the second account, create another S3 bucket to receive data from the radar station with the most accurate data. Set up a new replication rule for this new S3 bucket to separate the replication from the other radar stations. Monitor the maximum replication time to the destination. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to trigger an alert when the time exceeds the desired threshold.
C. Enable Amazon S3 Transfer Acceleration on the source S3 bucket, and configure the radar station with the most accurate data to use the new endpoint. Monitor the S3 destination bucket's TotalRequestLatency metric. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to trigger an alert if this status changes.
D. Create a new S3 replication rule on the source S3 bucket that filters for the keys that use the prefix of the radar station with the most accurate data. Enable S3 Replication Time Control (S3 RTC). Monitor the maximum replication time to the destination. Create an Amazon EventBridge (Amazon CloudWatch Events) rule to trigger an alert when the time exceeds the desired threshold.



# SAP-C02 Analysis — Amazon S3 Replication Time Control (S3 RTC)

## Understanding the Problem

The scenario focuses on:

- Cross-account Amazon S3 replication
- Compliance-driven replication requirements
- Time-critical scientific/radar data
- Monitoring replication completion within 30 minutes
- Replication only needs strict SLA monitoring for one specific prefix (one radar station)

Key clues:

- Data already uses S3 replication
- Objects are organized by prefix
- Requirement is specifically:
  - "must be monitored for completion within 30 minutes"
- One radar station has priority
- Need deterministic replication SLA visibility

This is a classic use case for:
- Amazon S3 Replication Time Control (S3 RTC)

---

# Correct Option

## D. Create a dedicated replication rule for the radar station prefix and enable S3 RTC

### Why this is correct

This option directly aligns with AWS-native replication SLA features.

The architecture:

1. Create a replication rule scoped to:
   - the radar station prefix
2. Enable:
   - S3 Replication Time Control (RTC)
3. Monitor:
   - replication metrics
4. Alert:
   - via EventBridge/CloudWatch when thresholds exceed limits

---

## Key AWS Feature: S3 Replication Time Control (RTC)

S3 RTC provides:

- Replication SLA backed by AWS
- 99.99% of new objects replicated within 15 minutes
- Replication metrics and events
- Compliance-oriented monitoring

This is specifically designed for:
- financial systems
- medical systems
- mission-critical ingestion pipelines
- regulated workloads

The exam expects recognition that:
- ordinary S3 replication is asynchronous without guaranteed timing
- RTC adds measurable replication guarantees

---

## Why Prefix Filtering Matters

The requirement applies only to:
- one radar station

The objects are already:
- prefixed by radar station ID

Therefore:
- create a replication rule targeting only that prefix

Benefits:
- lower RTC cost
- focused monitoring
- avoids enabling RTC globally unnecessarily

---

## Monitoring Components

S3 RTC integrates with:
- Amazon CloudWatch metrics
- Event notifications
- EventBridge

Metrics include:
- `BytesPendingReplication`
- `ReplicationLatency`
- `OperationsPendingReplication`

You can create:
- alarms when replication exceeds thresholds

# Incorrect Options

---

## A. AWS DataSync

### Why incorrect

AWS DataSync is not the appropriate service here.

Problems:

- Requirement already uses S3 replication
- DataSync is designed for:
  - migrations
  - scheduled transfers
  - hybrid movement
  - NFS/EFS/FSx/object migration
- DataSync does not provide:
  - S3 replication SLA guarantees
  - native replication-time compliance monitoring
- "TRANSFERRING status" does not indicate:
  - object-level compliance replication completion

Important exam distinction:

| Service | Purpose |
|---|---|
| S3 Replication | Continuous object replication |
| S3 RTC | SLA-backed replication |
| DataSync | Bulk/scheduled transfer |

This option distracts candidates with "monitoring" language.

---

## B. Create another bucket and replication rule

### Why incorrect

This introduces unnecessary infrastructure complexity.

Issues:

- Additional bucket is unnecessary
- Requirement is about:
  - replication timing guarantees
  - not data separation
- Simply monitoring "maximum replication time" without RTC:
  - does NOT guarantee SLA
- Standard replication:
  - has no deterministic completion guarantee

The key exam trap:
- Monitoring alone is not enough
- You need the AWS feature that provides replication guarantees

---

## C. S3 Transfer Acceleration

### Why incorrect

S3 Transfer Acceleration helps:
- uploads into S3 over long geographic distances

It does NOT help:
- replication between buckets

Critical distinction:

| Feature | Solves |
|---|---|
| Transfer Acceleration | Faster client uploads to S3 |
| S3 RTC | Faster monitored replication between buckets |

Also:
- `TotalRequestLatency` is unrelated to replication completion SLA

This option mixes:
- ingestion optimization
with
- replication monitoring

They are separate concerns.

# Enterprise Hands-On Knowledge

## Real-World Pattern in Medical/Industrial Systems

In highly regulated environments:

- scientific telemetry
- medical imaging
- satellite/radar systems
- manufacturing sensors

organizations often use:

- S3 Cross-Region Replication (CRR)
- Cross-account replication
- RTC-enabled prefixes for critical datasets

Example pattern:

```text
s3://mission-data/
    radar-001/
    radar-002/
    radar-003-critical/
```

Replication rules:

| Prefix | RTC |
|---|---|
| radar-001 | No |
| radar-002 | No |
| radar-003-critical | Yes |

This minimizes:
- RTC
- replication overhead

while maintaining:
- compliance for critical telemetry

## RTC Cost Awareness
