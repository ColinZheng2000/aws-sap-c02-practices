A solutions architect is reviewing a company's process for taking snapshots of Amazon RDS DB instances. The company takes automatic snapshots every day and retains the snapshots for 7 days.

The solutions architect needs to recommend a solution that takes snapshots every 6 hours and retains the snapshots for 30 days. The company uses AWS Organizations to manage all of its AWS accounts. The company needs a consolidated view of the health of the RDS snapshots.

Which solution will meet these requirements with the LEAST operational overhead?

A. Turn on the cross-account management feature in AWS Backup. Create a backup plan that specifies the frequency and retention requirements. Add a tag to the DB instances. Apply the backup plan by using tags. Use AWS Backup to monitor the status of the backups.
B. Turn on the cross-account management feature in Amazon RDS. Create a snapshot global policy that specifies the frequency and retention requirements. Use the RDS console in the management account to monitor the status of the backups.
C. Turn on the cross-account management feature in AWS CloudFormation. From the management account, deploy a CloudFormation stack set that contains a backup plan from AWS Backup that specifies the frequency and retention requirements. Create an AWS Lambda function in the management account to monitor the status of the backups. Create an Amazon EventBridge rule in each account to run the Lambda function on a schedule.
D. Configure AWS Backup in each account. Create an Amazon Data Lifecycle Manager lifecycle policy that specifies the frequency and retention requirements. Specify the DB instances as the target resource Use the Amazon Data Lifecycle Manager console in each member account to monitor the status of the backups.



Suggest Answer: A

---

## 📊 Analysis

### 🔍 Understanding the Problem

A company currently uses **automatic Amazon RDS snapshots** (daily, 7-day retention). They need to upgrade this to **every 6 hours with 30-day retention**. The environment spans **multiple AWS accounts managed via AWS Organizations**, and the solutions architect must provide a **consolidated view** of snapshot health across all accounts. The solution must have the **least operational overhead**.

Key requirements:
- Increase snapshot frequency from daily → every 6 hours
- Increase retention from 7 days → 30 days
- Cross-account, centralized management via AWS Organizations
- Consolidated monitoring/visibility of all backups
- Minimal operational effort (automation preferred)

### ✅ Correct Option: A — AWS Backup with cross-account management, tag-based backup plan, AWS Backup monitoring

**Why it's correct:**

**AWS Backup** is the fully managed, centralized backup service that directly addresses all requirements:

1. **Cross-Account Management**: AWS Backup natively supports cross-account backup management within AWS Organizations. You enable this feature from the **management account** (formerly master account), allowing a single backup policy to govern backups across all member accounts — no per-account configuration needed.

2. **Backup Plan Flexibility**: Backup plans in AWS Backup let you define:
   - **Frequency**: Supports cron expressions — `cron(0 0/6 * * ? *)` for every 6 hours
   - **Retention**: 30-day retention rule easily configured
   - **Lifecycle**: Can also transition to cold storage for longer retention if needed

3. **Tag-Based Resource Assignment**: Instead of manually selecting each RDS instance across dozens of accounts, you tag RDS instances (e.g., `Backup:Required`) and the backup plan automatically discovers and applies to tagged resources across all accounts. This is massively scalable.

4. **Consolidated Monitoring**: AWS Backup provides a **centralized dashboard** showing backup job status, compliance, and health across all accounts. You can also use AWS Backup Audit Manager for compliance reporting.

5. **Least Operational Overhead**: Once configured, the backup plan runs automatically across all accounts. No Lambda functions to maintain, no EventBridge rules to manage, no CloudFormation stacks to update. Pure service-native automation.

**Official AWS Documentation**: [AWS Backup Cross-Account Management](https://docs.aws.amazon.com/aws-backup/latest/devguide/cross-account-management.html)

### ❌ Incorrect Options Explanation

**B. Turn on the cross-account management feature in Amazon RDS. Create a snapshot global policy.**

- **Why incorrect**: Amazon RDS does **not have** a "cross-account management feature" or a "snapshot global policy" mechanism. These are **fabricated features**. RDS snapshots are managed per-instance, per-region, per-account. While you can share RDS snapshots across accounts manually, there is no native centralized policy engine for RDS snapshots across an organization. RDS alone cannot meet the cross-account consolidated management requirement without building custom automation.

**C. AWS CloudFormation StackSets + AWS Backup + Lambda + EventBridge**

- **Why incorrect**: This is a **Rube Goldberg machine** — over-engineered and operationally heavy:
  - CloudFormation StackSets deploy the backup plan to each account (redundant — AWS Backup already supports cross-account natively)
  - Custom Lambda function for monitoring status (redundant — AWS Backup has built-in consolidated monitoring)
  - EventBridge rules in each account to trigger Lambda (maintenance burden across N accounts)
  - This introduces ongoing operational overhead: Lambda code maintenance, EventBridge rule management, StackSet drift detection, IAM role complexities across accounts
  - The question explicitly asks for **LEAST operational overhead** — this is the opposite

**D. AWS Backup in each account + Amazon Data Lifecycle Manager + per-account monitoring**

- **Why incorrect**: 
  - **Amazon Data Lifecycle Manager (DLM)** manages **EBS snapshots** and **EBS-backed AMIs**, not RDS snapshots. Using DLM for RDS is technically impossible — wrong service for the resource type.
  - Configuring AWS Backup **individually in each account** defeats the consolidation requirement. Each account would need its own backup plan, and monitoring would require logging into each account separately — directly violates the "consolidated view" requirement.
  - High operational overhead: N accounts × manual configuration + N consoles to check

### 🏭 Enterprise Hands-On Knowledge

1. **AWS Backup vs. Native Service Snapshots**:
   | Feature | AWS Backup | RDS Native Snapshots |
   |---|---|---|
   | Cross-account | ✅ Native support | ❌ Manual share only |
   | Cross-region | ✅ Built-in copy | ❌ Manual copy |
   | Centralized monitoring | ✅ Dashboard + Audit Manager | ❌ Per-instance only |
   | Backup policies | ✅ Plans + tag-based | ❌ Manual scheduling |
   | Compliance reporting | ✅ AWS Backup Audit Manager | ❌ Custom required |
   | Vault lock (WORM) | ✅ Backup Vault Lock | ❌ N/A |

2. **AWS Organizations + AWS Backup Integration**: When you enable cross-account management in AWS Backup, the management account's backup policies automatically apply to all accounts in the organization (or selected OUs). This uses AWS Organizations' trusted access capability. The delegated administrator account (if configured) can manage backups across the organization without needing access to the management account's root credentials.

3. **Backup Plan Tag-Based Assignment Gotcha**: Tags are case-sensitive. A common pitfall is tagging an RDS instance with `backup:required` while the backup plan looks for `Backup:Required`. Use AWS Tag Editor or AWS Config rules to validate tag compliance across your fleet.

4. **RDS Snapshot Automation Without AWS Backup**: If you must use native RDS snapshots (e.g., for very specific RDS engine features), you would need AWS Lambda + Amazon EventBridge Scheduler (or CloudWatch Events) to invoke `rds:CreateDBSnapshot` and `rds:DeleteDBSnapshot` APIs. This is fundamentally more operational overhead than using AWS Backup.

5. **Cost Consideration**: AWS Backup charges per backup storage (GB-month) and for cross-region data transfer. For RDS, backup storage beyond the free automated backup window (matching your allocated storage) incurs charges. AWS Backup consolidated billing makes cost allocation simpler across accounts.

6. **Compliance & Audit**: AWS Backup Audit Manager provides pre-built compliance frameworks (e.g., NIST, GDPR) and can generate automated audit reports showing backup compliance across all accounts — critical for medical/industrial companies subject to regulatory requirements.
