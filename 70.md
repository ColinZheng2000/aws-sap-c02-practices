A company wants to change its internal cloud billing strategy for each of its business units. Currently, the cloud governance team shares reports for overall cloud spending with the head of each business unit. The company uses AWS Organizations to manage the separate AWS accounts for each business unit. The existing tagging standard in Organizations includes the application, environment, and owner. The cloud governance team wants a centralized solution so each business unit receives monthly reports on its cloud spending. The solution should also send notifications for any cloud spending that exceeds a set threshold.  
Which solution is the MOST cost-effective way to meet these requirements?  

- A. Configure AWS Budgets in each account and configure budget alerts that are grouped by application, environment, and owner. Add each business unit to an Amazon SNS topic for each alert. Use Cost Explorer in each account to create monthly reports for each business unit.
- B. Configure AWS Budgets in the organization's master account and configure budget alerts that are grouped by application, environment, and owner. Add each business unit to an Amazon SNS topic for each alert. Use Cost Explorer in the organization's master account to create monthly reports for each business unit.
- C. Configure AWS Budgets in each account and configure budget alerts that are grouped by application, environment, and owner. Add each business unit to an Amazon SNS topic for each alert. Use the AWS Billing and Cost Management dashboard in each account to create monthly reports for each business unit.
- D. Enable AWS Cost and Usage Reports in the organization's master account and configure reports grouped by application, environment, and owner. Create an AWS Lambda function that processes AWS Cost and Usage Reports, sends budget alerts, and sends monthly reports to each business unit's email list.

## Analysis

### Understanding the Problem

The company uses **AWS Organizations** to manage multiple accounts (one per business unit) and has a defined **tagging standard** (application, environment, owner) for resource allocation. The goal is to establish a **centralized, cost-effective** solution to:

1. Provide **monthly reports** on cloud spending _per business unit_.
    
2. Send **notifications** when spending **exceeds a set threshold** (budget alerts).
    

The key requirements are **centralization** (to manage across all accounts easily from the governance team) and **cost-effectiveness**. Since the business units are managed under AWS Organizations, the master (management) account is the logical central point for governance and reporting. Cost reporting should leverage the existing tagging standard to break down costs accurately.

### Services Explanation

| **Service**                                  | **Description**                                                                                                                                                                                                  | **Relevance to Problem**                                                                                                                                                                                               |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **AWS Organizations**                        | Account management service that enables you to consolidate multiple AWS accounts into an organization that you create and centrally manage. **Key for centralized governance and consolidated billing.**         | Essential foundation for the multi-account structure. Consolidated billing is key for centralized reporting.                                                                                                           |
| **AWS Budgets**                              | Allows you to set custom budgets to track your costs and usage. You can set up alerts when your costs or usage exceed (or are forecasted to exceed) your budgeted amount. **Alerts can be sent via Amazon SNS.** | Directly meets the requirement for **notifications for exceeding a threshold**.                                                                                                                                        |
| **Cost Explorer**                            | Provides a graphical interface for viewing, understanding, and analyzing your AWS costs and usage over time. **Can filter by tags and linked accounts (business units).**                                        | Useful for creating and viewing ad-hoc or saved cost reports, but **doesn't automate monthly report delivery**.                                                                                                        |
| **AWS Cost and Usage Reports (CUR)**         | The most comprehensive set of AWS cost and usage data available. **Can be configured to include resource tags** and delivered to an S3 bucket in the payer account.                                              | Provides the raw, detailed data needed for **custom, automated reporting and analysis**, including spending breakdown by tags. **Essential for the most flexible and potentially cost-effective automated reporting.** |
| **Amazon SNS (Simple Notification Service)** | A managed service that provides message delivery from publishers to subscribers. **Used by AWS Budgets for sending alerts.**                                                                                     | Essential component for sending budget **notifications**.                                                                                                                                                              |
| **AWS Lambda**                               | A serverless compute service that runs code in response to events. **Can be triggered on CUR delivery and process the data.**                                                                                    | Can be used to **automate custom report generation** (by processing CUR data) and **send notifications/reports** via email or other channels.                                                                          |

### Correct Option(s) Explanation

**B. Configure AWS Budgets in the organization's master account and configure budget alerts that are grouped by application, environment, and owner. Add each business unit to an Amazon SNS topic for each alert. Use Cost Explorer in the organization's master account to create monthly reports for each business unit.**

- **Centralized Control:** Configuring **AWS Budgets in the master account** allows the cloud governance team to manage all alerts and costs across all linked business unit accounts from a **single, central place**. This directly addresses the **centralized solution** requirement.
    
- **Threshold Notification:** **AWS Budgets** directly meets the requirement for setting a **threshold and sending notifications** via Amazon SNS.
    
- **Cost Breakdown:** Budgets can be configured to track costs **grouped by the existing tags** (application, environment, owner) and for specific **linked accounts** (business units), ensuring accurate cost allocation.
    
- **Monthly Reports:** **Cost Explorer** in the master account can show aggregated and filtered cost data across all accounts and tags, and its reports can be saved and shared, fulfilling the **monthly reports** requirement.
    
- **Cost-Effectiveness:** **AWS Budgets and Cost Explorer are available at no additional charge** (you only pay for the underlying services). This is the **MOST cost-effective** built-in, centralized solution that directly meets all stated requirements without needing custom code (like in option D).
    

---

### Incorrect Options Explanation

**A. Configure AWS Budgets in each account... Use Cost Explorer in each account...**

- **Not Centralized:** Requires configuration and management in _each individual account_, which is not a **centralized solution** and is inefficient for the cloud governance team.
    

**C. Configure AWS Budgets in each account... Use the AWS Billing and Cost Management dashboard in each account...**

- **Not Centralized:** Similar to A, managing Budgets and accessing the Billing dashboard _in each account_ is not a **centralized solution**.
    
- **Limited Reporting:** The Billing and Cost Management dashboard is less flexible and powerful for creating detailed, tag-based reports compared to Cost Explorer.
    

**D. Enable AWS Cost and Usage Reports in the organization's master account... Create an AWS Lambda function that processes AWS Cost and Usage Reports, sends budget alerts, and sends monthly reports...**

- **Not the MOST Cost-Effective:** While this solution is centralized and highly flexible (it uses the comprehensive CUR data and a Lambda function for automation), it introduces **operational overhead** and **additional costs** for developing, testing, deploying, and maintaining the custom AWS Lambda function, along with storage costs for the CUR files in Amazon S3.
    
- The requirement for **budget alerts** and **monthly reports** can be met using the native, **free-of-charge** services (AWS Budgets and Cost Explorer) as described in option B. Therefore, option D is not the **MOST cost-effective** way, even though it is a valid, highly-automated technical solution.
