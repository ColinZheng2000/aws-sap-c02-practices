A company is planning to migrate an on-premises data center to AWS. The company currently hosts the data center on Linux-based VMware VMs. A solutions architect must collect information about network dependencies between the VMs. The information must be in the form of a diagram that details host IP addresses, hostnames, and network connection information.

Which solution will meet these requirements?

A. Use AWS Application Discovery Service. Select an AWS Migration Hub home AWS Region. Install the AWS Application Discovery Agent on the on-premises servers for data collection. Grant permissions to Application Discovery Service to use the Migration Hub network diagrams.
B. Use the AWS Application Discovery Service Agentless Collector for server data collection. Export the network diagrams from the AWS Migration Hub in .png format.
C. Install the AWS Application Migration Service agent on the on-premises servers for data collection. Use AWS Migration Hub data in Workload Discovery on AWS to generate network diagrams.
D. Install the AWS Application Migration Service agent on the on-premises servers for data collection. Export data from AWS Migration Hub in .csv format into an Amazon CloudWatch dashboard to generate network diagrams.



# AWS SAP-C02 Question Analysis — Q339

## 📋 Analysis

### 1. Understanding the Problem

The company is preparing for a large-scale VMware migration from an on-premises Linux environment to AWS. Before migration execution, the solutions architect must perform application discovery and dependency mapping.

The required output is:

- Network dependency relationships between VMs
- Hostnames and IP addresses
- Connection-level visibility
- Diagram-style visualization

This requirement belongs to the migration assessment and discovery phase, not the migration execution phase.

The core AWS services commonly involved are:

- AWS Application Discovery Service (ADS)
- AWS Migration Hub

A critical SAP-C02 exam skill is distinguishing:

- Discovery/assessment services
vs.
- Migration replication services

The question specifically emphasizes dependency visualization, which strongly points to AWS Application Discovery Service.

- --

### 2. Options Analysis

#### ✅ Option A (CORRECT): Use AWS Application Discovery Service agents with Migration Hub network diagrams

This is the correct answer because it directly fulfills the dependency mapping and visualization requirements.

Why this works:

- AWS Application Discovery Agent collects:
  - Hostnames
  - IP addresses
  - Running processes
  - Network connection metadata
  - Application dependencies
- AWS Migration Hub integrates with discovery data
- Migration Hub network diagrams visualize communication flows between servers

Most important technical detail:

- The agent-based discovery approach provides deep process and network dependency visibility
- This is exactly what enterprise migration planning requires

Enterprise migration reality:

In medical, manufacturing, and industrial environments, many applications are poorly documented. Hidden dependencies often exist between:

- Legacy middleware
- Database servers
- Shared storage systems
- Batch-processing hosts
- Internal APIs

Without dependency discovery:
- Migration waves become risky
- Firewall changes may fail
- Cutovers can cause outages

Why AWS recommends agent-based discovery:
- More accurate dependency mapping
- Better application grouping
- Better migration sequencing

Official AWS references:
- AWS Application Discovery Service User Guide
- AWS Migration Hub Network Visualization
- AWS Prescriptive Guidance for Migration Discovery

- --

#### ❌ Option B (INCORRECT): Agentless discovery is insufficient for detailed dependency mapping

This option appears reasonable at first glance because the AWS Application Discovery Service Agentless Collector supports VMware environments.

However, it is still incorrect.

Why:

- Agentless Collector mainly gathers:
  - VMware inventory
  - VM configurations
  - Performance metrics
- It does not provide the same depth of process-level network dependency information as the Discovery Agent

The requirement explicitly asks for:
- Network dependencies
- Connection information
- Detailed topology diagrams

Those requirements are better addressed by agent-based discovery.

Additional issue:

- Exporting diagrams as PNG is not the architectural requirement
- The key requirement is accurate dependency discovery itself

Real-world enterprise practice:

Large enterprises often combine both:
- Agentless collection for broad inventory discovery
- Agent-based collection for critical application dependency analysis

SAP-C02 usually expects the most complete technical solution.

- --

#### ❌ Option C (INCORRECT): AWS Application Migration Service is for replication, not dependency discovery

This option confuses migration execution with migration assessment.

AWS Application Migration Service (MGN) is designed for:
- Lift-and-shift migrations
- Block-level replication
- Test cutovers
- Final server migration

It is not designed to:
- Build dependency maps
- Generate topology diagrams
- Perform deep discovery analysis

Another issue:
- Workload Discovery on AWS is unrelated to native MGN dependency visualization workflows

This is a classic SAP exam distractor:
- Using a migration service when the requirement is actually discovery and assessment

Correct architectural distinction:

| Purpose | AWS Service |
| --- | --- |
| Discovery & dependency mapping | AWS Application Discovery Service |
| Server replication & cutover | AWS Application Migration Service |

- --

#### ❌ Option D (INCORRECT): CloudWatch dashboards cannot generate dependency topology diagrams

This option combines unrelated services into an unsupported design.

Why it fails:

- AWS Migration Hub data is not visualized through CloudWatch dashboards for dependency mapping
- CloudWatch dashboards focus on:
  - Metrics
  - Logs
  - Alarms
  - Operational monitoring
- CloudWatch does not provide application dependency graph generation

Another problem:
- AWS Application Migration Service still does not solve the discovery requirement

Architectural anti-pattern:

Using monitoring dashboards as topology mapping systems is incorrect because:
- Monitoring telemetry ≠ dependency discovery
- Metrics visualization ≠ network relationship mapping

This is not an AWS-supported migration discovery architecture.

### 3. Hands-On Knowledge

In enterprise migration projects, discovery typically occurs in multiple stages.

Typical migration discovery workflow:

#### Phase 1 — Broad Infrastructure Inventory
Use Agentless Collector to gather:
- VMware inventory
- VM sizing
- CPU/memory utilization
- OS versions

Benefit:
- Rapid discovery with minimal operational impact

- --

#### Phase 2 — Deep Dependency Discovery
Install Discovery Agents on:
- Critical application servers
- Middleware nodes
- Database hosts

Collected information includes:
- TCP connection flows
- Running processes
- Inter-server communication
- Application dependencies

This phase identifies:
- Hidden application coupling
- Legacy communication patterns
- Migration wave dependencies

- --

#### Phase 3 — Migration Wave Planning
Use Migration Hub insights to:
- Group applications
- Define migration batches
- Reduce downtime risk

Typical enterprise migration grouping:
- Wave 1: Stateless services
- Wave 2: Middleware services
- Wave 3: Databases and tightly coupled systems

### Common Enterprise Gotchas

#### Hidden Dependencies
Very common in legacy environments:
- Hardcoded IP addresses
- Shared NFS mounts
- LDAP dependencies
- Batch jobs
- Old Oracle listener configurations

These are often undocumented.

- --

#### Discovery Duration
Best practice:
- Run discovery agents for at least 2 weeks
- Capture weekly and monthly traffic patterns

Otherwise:
- Rare but critical dependencies may be missed

- --

#### Security Team Coordination
Discovery agents require:
- Firewall approvals
- IAM permissions
- Outbound connectivity validation

Medical and industrial companies often require:
- Change management approval
- Security review
- Compliance validation

### Useful AWS CLI Commands

List discovered servers:
```bash
aws discovery describe-configurations \
  - -configuration-type SERVER
```

Export discovery data:
```bash
aws discovery export-configurations
```

Describe export status:
```bash
aws discovery describe-export-configurations
```

1. "What is the difference between AWS Application Discovery Service and AWS Application Migration Service?"
   → 回答要点 (Key Answer Points)：AWS Application Discovery Service is used during assessment and planning to collect infrastructure inventory and application dependency information. AWS Application Migration Service is used later to replicate and migrate servers into AWS with minimal downtime.

2. "When should you use agentless discovery versus agent-based discovery?"
   → 回答要点 (Key Answer Points)：Agentless discovery is useful for quickly inventorying VMware environments with low operational overhead. Agent-based discovery is preferred when deep dependency mapping, process visibility, and accurate network communication analysis are required.
