A solutions architect must analyze a company's Amazon EC2 instances and Amazon Elastic Block Store (Amazon EBS) volumes to determine whether the company is using resources efficiently. The company is running several large, high-memory EC2 instances to host database clusters that are deployed in active/ passive configurations. The utilization of these EC2 instances varies by the applications that use the databases, and the company has not identified a pattern.
The solutions architect must analyze the environment and take action based on the findings.
Which solution meets these requirements MOST cost-effectively?

A. Create a dashboard by using AWS Systems Manager OpsCenter. Configure visualizations for Amazon CloudWatch metrics that are associated with the EC2 instances and their EBS volumes. Review the dashboard periodically, and identify usage patterns. Rightsize the EC2 instances based on the peaks in the metrics.
B. Turn on Amazon CloudWatch detailed monitoring for the EC2 instances and their EBS volumes. Create and review a dashboard that is based on the metrics. Identify usage patterns. Rightsize the EC2 instances based on the peaks in the metrics.
C. Install the Amazon CloudWatch agent on each of the EC2 instances. Turn on AWS Compute Optimizer, and let it run for at least 12 hours. Review the recommendations from Compute Optimizer, and rightsize the EC2 instances as directed.
D. Sign up for the AWS Enterprise Support plan. Turn on AWS Trusted Advisor. Wait 12 hours. Review the recommendations from Trusted Advisor, and rightsize the EC2 instances as directed.


# Q: Cost-effective analysis and rightsizing of EC2 & EBS usage

## 🧠 Understanding the Problem
- Company runs **large, high-memory EC2 instances** hosting **database clusters** in **active/passive** mode.
- Utilization varies by application; **no clear usage pattern is known**.
- Goal: **Analyze** current usage of EC2 and EBS and then **take rightsizing actions**.
- Must be **MOST cost-effective**:
  - Avoid unnecessary manual effort.
  - Avoid extra paid services if there is a cheaper/built-in alternative.
  - Use a tool that can **analyze utilization and provide concrete recommendations**.

Key idea: We need an AWS-native way to:
1. Collect utilization metrics.
2. Analyze them.
3. Provide **automated rightsizing recommendations** for EC2 (and ideally EBS).
4. Do so **cost-effectively**.

- --

## 🧩 Services Explanation Relevant to Options

### 1. Amazon CloudWatch
- **Service type**: Monitoring & observability.
- **Basic vs detailed monitoring**:
  - Basic: 5-minute granularity for EC2 metrics, included at no extra EC2 charge.
  - Detailed: 1-minute granularity, **extra cost** per instance.
- Use cases:
  - Dashboards for visualization.
  - Alarms, log analysis.
- Limitations:
  - CloudWatch **does not generate rightsizing recommendations** by itself.

Docs:  
- https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/WhatIsCloudWatch.html

### 2. AWS Systems Manager OpsCenter
- **Service type**: Part of Systems Manager for **operations issues (OpsItems)**.
- Provides:
  - Central place to view, investigate, and resolve operational issues.
  - Can embed CloudWatch metrics/graphs for affected resources.
- Not a rightsizing engine; its goal is operational incident management, not optimization.

Docs:  
- https://docs.aws.amazon.com/systems-manager/latest/userguide/OpsCenter.html

### 3. AWS Compute Optimizer
- **Service type**: **Cost optimization & performance recommendations**.
- Uses machine learning and historical CloudWatch metrics to recommend:
  - **EC2 instance** rightsizing: instance families & sizes.
  - **EBS volume** types and sizes.
  - Lambda & Auto Scaling group resources.
- Requirements:
  - Needs CloudWatch metrics. Standard monitoring is sufficient; does **not require detailed monitoring**.
  - Needs **a period of metrics history** (usually **at least 12 hours**, better with days/weeks for accuracy).
- Cost:
  - Basic recommendations for EC2, EBS, Lambda, and ASG are generally **included at no extra cost** (check up-to-date pricing).
- Perfectly aligned with:
  - Analyzing **efficiency**.
  - Automatically suggesting **rightsizing**.

Docs:  
- https://docs.aws.amazon.com/compute-optimizer/latest/ug/what-is-compute-optimizer.html  
- https://docs.aws.amazon.com/compute-optimizer/latest/ug/getting-started.html

### 4. AWS Trusted Advisor
- **Service type**: Guidance for **cost optimization, performance, security, fault tolerance, and service limits**.
- With **Business/Enterprise support**, you get **full checks**, including:
  - **Cost optimization** checks (underutilized EC2, idle load balancers, etc.)
- However:
  - Requires **Business or Enterprise support** plan (significant cost).
  - Recommendations are **less granular** and **less ML-driven** than Compute Optimizer.
  - Not the most cost-effective way solely for rightsizing.

Docs:  
- https://docs.aws.amazon.com/awssupport/latest/user/trusted-advisor.html  
- Support plan pricing: https://aws.amazon.com/premiumsupport/pricing/

## ✅ Correct Option: C

> **C. Install the Amazon CloudWatch agent on each of the EC2 instances. Turn on AWS Compute Optimizer, and let it run for at least 12 hours. Review the recommendations from Compute Optimizer, and rightsize the EC2 instances as directed.**

### Why C is the MOST cost-effective and aligned solution

1. **Automated analysis & recommendations**  
   - AWS **Compute Optimizer** is purpose-built to analyze **EC2 instance utilization and EBS volumes** and provide **rightsizing recommendations** based on actual historical usage.
   - It considers **CPU, memory, network, disk**, etc., where data is available.

2. **No need for manual pattern discovery**  
   - The problem states there is **no known pattern**, and usage varies.  
   - Instead of manually inspecting metrics, Compute Optimizer uses ML to discover patterns and propose optimal instance types/sizes.

3. **Cost-effectiveness**  
   - Compute Optimizer is typically **included** (for core resources) and uses **existing CloudWatch metrics**.
   - No requirement for expensive support plans.
   - The CloudWatch agent:
     - Enables **detailed system-level metrics** (including memory and disk I/O at OS-level), which improves recommendation accuracy.
     - Agent cost is mainly the cost of additional metrics in CloudWatch, which is relatively low compared with:
       - Overprovisioned high-memory instances.
       - Enterprise Support fee.

4. **Action-oriented**  
   - The question: “analyze the environment and **take action** based on the findings.”  
   - Compute Optimizer produces direct actionable recommendations (e.g., “m5.4xlarge → r5.2xlarge or r6i.2xlarge”), making it well-suited to support rightsizing decisions.

5. **Time horizon**  
   - Letting it run **at least 12 hours** is the minimal requirement, as stated. Longer gives better recommendations, but 12h is consistent with Compute Optimizer guidance for initial results.

**Key doc reference:**  
AWS Compute Optimizer “What is” and getting started pages confirm that:
- It uses CloudWatch metrics.
- It generates **rightsizing** recommendations:  
https://docs.aws.amazon.com/compute-optimizer/latest/ug/what-is-compute-optimizer.html

## ❌ Insightful Explanation of Incorrect Options

### A. Systems Manager OpsCenter + CloudWatch dashboard

> **A. Create a dashboard by using AWS Systems Manager OpsCenter. Configure visualizations for Amazon CloudWatch metrics that are associated with the EC2 instances and their EBS volumes. Review the dashboard periodically, and identify usage patterns. Rightsize the EC2 instances based on the peaks in the metrics.**

**Why it’s not the best:**
- **OpsCenter is for operational issues**, not cost optimization.  
  - It can show metrics but doesn’t analyze or recommend instance types.
- Requires **manual**:
  - Building dashboards.
  - Periodically reviewing.
  - Manually interpreting metrics and deciding on rightsizing.
- This manual process is:
  - Error-prone.
  - Labor-intensive.
  - Not “most cost-effective” when a **free, automated tool (Compute Optimizer)** exists.

**Useful for exam knowledge:**
- Understand OpsCenter is for **OpsItems / incidents**, not optimization or rightsizing.  
Docs: https://docs.aws.amazon.com/systems-manager/latest/userguide/OpsCenter.html

- --

### B. Turn on CloudWatch detailed monitoring & analyze dashboards

> **B. Turn on Amazon CloudWatch detailed monitoring for the EC2 instances and their EBS volumes. Create and review a dashboard that is based on the metrics. Identify usage patterns. Rightsize the EC2 instances based on the peaks in the metrics.**

**Why it’s not the best:**
- **Detailed monitoring** costs extra per instance and only improves metric granularity (1-minute).
- Like option A, this relies on **manual analysis** of charts and patterns:
  - You must infer when to scale up/down and to what size.
- Does not leverage **Compute Optimizer’s ML-based recommendations**.
- “Most cost-effective” is about:
  - Reducing human effort.
  - Avoiding paid extras where not needed.
- Compute Optimizer already uses standard metrics and doesn’t require detailed monitoring to function.

**Important nuance:**
- For many rightsizing tasks, 5-minute metrics are sufficient.
- Detailed monitoring is often used for fast-reacting autoscaling, not for long-term rightsizing.

Docs:  
- CloudWatch detailed monitoring: https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-cloudwatch-new.html

- --

### D. Enterprise Support + Trusted Advisor

> **D. Sign up for the AWS Enterprise Support plan. Turn on AWS Trusted Advisor. Wait 12 hours. Review the recommendations from Trusted Advisor, and rightsize the EC2 instances as directed.**

**Why it’s not the most cost-effective:**
- **Enterprise Support** is very expensive (percentage of AWS usage).
- You do get full **Trusted Advisor** checks including cost optimization, but:
  - Trusted Advisor checks are **higher level**, such as:
    - Low utilization EC2 instances.
    - Idle load balancers.
    - Underutilized RDS instances.
  - It does not perform the same ML-based, granular rightsizing analysis as **Compute Optimizer**.
- For the specific task “analyze EC2 and EBS utilization and rightsize,” paying for the Enterprise Support tier solely for this is **not cost-effective**.

Docs:  
- Trusted Advisor: https://docs.aws.amazon.com/awssupport/latest/user/trusted-advisor.html  
- Support pricing: https://aws.amazon.com/premiumsupport/pricing/

## 🔁 Summary

- The problem asks: analyze EC2 & EBS usage and **rightsize** most cost-effectively.
- **Compute Optimizer** is the native AWS service designed exactly for **rightsizing analysis** based on historical metrics.
- Installing the **CloudWatch agent** enables deeper metrics (such as memory usage) that enhance rightsizing accuracy.
- **Option C** uses these tools correctly and avoids unnecessary manual work and expensive Enterprise Support.

> ✅ **Correct answer: C**
