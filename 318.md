A company is creating a solution that can move 400 employees into a remote working environment in the event of an unexpected disaster. The user desktops have a mix of Windows and Linux operating systems. Multiple types of software, such as web browsers and mail clients, are installed on each desktop.

A solutions architect needs to implement a solution that can be integrated with the company’s on-premises Active Directory to allow employees to use their existing identity credentials. The solution must provide multifactor authentication (MFA) and must replicate the user experience from the existing desktops.

Which solution will meet these requirements?

A. Use Amazon WorkSpaces for the cloud desktop service. Set up a VPN connection to the on-premises network. Create an AD Connector, and connect to the on-premises Active Directory. Activate MFA for Amazon WorkSpaces by using the AWS Management Console.
B. Use Amazon AppStream 2.0 as an application streaming service. Configure Desktop View for the employees. Set up a VPN connection to the on-premises network. Set up Active Directory Federation Services (AD FS) on premises. Connect the VPC network to AD FS through the VPN connection.
C. Use Amazon WorkSpaces for the cloud desktop service. Set up a VPN connection to the on-premises network. Create an AD Connector, and connect to the on-premises Active Directory. Configure a RADIUS server for MFA.
D. Use Amazon AppStream 2.0 as an application streaming service. Set up Active Directory Federation Services on premises. Configure MFA to grant users access on AppStream 2.0.


# AWS SAP-C02 Question Analysis — Q318

## 📋 Analysis

### 1. Understanding the Problem

The company needs a disaster-response remote working solution for 400 employees.

Critical requirements:

- Employees currently use:
  - Windows desktops
  - Linux desktops
- Existing desktop experience must be replicated
- Existing software stack must remain available
- Integration with on-premises Active Directory
- Existing credentials must continue working
- MFA is mandatory
- Remote access during unexpected disasters

The most important keyword is:

> “replicate the user experience from the existing desktops”

This strongly implies:
- Full virtual desktop infrastructure (VDI)

NOT:
- Application streaming only

Therefore:
- Amazon WorkSpaces is the strongest candidate.

---

# 2. Core AWS Service Distinction

## Amazon WorkSpaces

Purpose:
- Managed cloud desktop service

Best for:
- Full desktop replacement
- Persistent user environments
- Enterprise remote workforce

Supports:
- Windows
- Linux
- Persistent desktops
- Existing applications
- Enterprise authentication

This directly matches the scenario.

---

## Amazon AppStream 2.0

Purpose:
- Application streaming service

Best for:
- Delivering individual applications
- Browser-based app access
- Non-persistent sessions

Although AppStream has Desktop View:
- SAP-C02 still positions it primarily as app streaming
- It is not the preferred answer for full desktop replacement scenarios

---

# 3. MFA Requirement — Critical Exam Detail

This is the key discriminator in the question.

For Amazon WorkSpaces:
- MFA requires:
  - RADIUS integration

This is a very common SAP-C02 exam trap.

Important:
- MFA is NOT enabled simply by clicking an option in the console.
- WorkSpaces integrates MFA through a RADIUS server.

Official AWS architecture:

```text
WorkSpaces
    ↓
Directory Service
    ↓
RADIUS MFA
```

Therefore:
- Any answer missing RADIUS is suspicious.

# 4. Option-by-Option Analysis

---

## ❌ Option A (INCORRECT)

> Use Amazon WorkSpaces...
> Activate MFA for Amazon WorkSpaces by using the AWS Management Console.

Why wrong:

Everything else is good:
- WorkSpaces
- VPN
- AD Connector

BUT:
- MFA implementation is inaccurate

WorkSpaces MFA requires:
- RADIUS integration

The console alone is insufficient.

This is a classic AWS certification precision trap.

---

## ❌ Option B (INCORRECT)

> Use Amazon AppStream 2.0 with Desktop View...

Problems:

- AppStream is primarily for application streaming
- Requirement asks for desktop replication
- Less suitable than WorkSpaces
- Adds unnecessary federation complexity

Even though AD FS can work:
- It is not the best architectural fit.

---

## ✅ Option C (CORRECT)

> Use Amazon WorkSpaces...
> Create an AD Connector...
> Configure a RADIUS server for MFA.

This fully satisfies every requirement.

### Why this works

#### Full desktop experience
- WorkSpaces provides persistent desktops
- Closely replicates physical desktops

#### Existing identity integration
- AD Connector connects to on-premises AD
- Users keep existing credentials

#### MFA support
- RADIUS is the required MFA mechanism

#### Remote disaster recovery workforce
- Employees can securely connect from anywhere
- Rapid cloud-based desktop provisioning

This is the AWS-recommended enterprise VDI design.

Official reference:
https://docs.aws.amazon.com/workspaces/latest/adminguide/multi-factor-authentication.html

---

## ❌ Option D (INCORRECT)

> Use AppStream 2.0...

Problems:

- Again focuses on application streaming
- Does not best replicate desktop environments
- Weaker fit for persistent desktop use cases

Even though MFA federation is possible:
- The service choice is still inferior to WorkSpaces.

# 5. Architecture Pattern

Typical enterprise deployment:

```text
Remote Employees
        ↓
Amazon WorkSpaces
        ↓
AD Connector
        ↓
On-Premises Active Directory
        ↓
RADIUS MFA Server
```

Connectivity:
- Site-to-Site VPN or Direct Connect between AWS and on-premises network

Authentication flow:
1. User enters AD credentials
2. AD Connector forwards authentication
3. RADIUS MFA challenge occurs
4. Desktop session launches

# 6. Why WorkSpaces Is Better Than AppStream Here

| Feature | WorkSpaces | AppStream 2.0 |
|---|---|---|
| Full desktop replacement | ✅ | Limited |
| Persistent desktops | ✅ | Usually no |
| User personalization | ✅ | Limited |
| Installed desktop software | ✅ | Partial |
| Best for remote workforce | ✅ | Sometimes |
| Primary use case | VDI | App streaming |

SAP-C02 shortcut:

- “Need a desktop” → WorkSpaces
- “Need an application” → AppStream

## 面试官可能追问

### 1. “WorkSpaces 和 AppStream 2.0 的区别是什么？”

回答要点：

WorkSpaces：
- 完整云桌面
- Persistent desktop
- 长期办公场景

AppStream 2.0：
- 应用流式传输
- 无需完整桌面
- 适合单应用交付

---

### 2. “为什么 WorkSpaces MFA 要用 RADIUS？”

回答要点：

- AWS WorkSpaces 通过 RADIUS 与企业 MFA 系统集成
- 可对接：
  - Duo
  - RSA
  - Azure MFA
  - Okta

RADIUS 是标准企业 MFA 集成方式。

---

### 3. “AD Connector 和 AWS Managed Microsoft AD 的区别？”

回答要点：

AD Connector：
- 代理到现有 AD
- 不存储目录数据
- 适合已有本地 AD

Managed Microsoft AD：
- AWS 托管目录
- 可独立运行
- 更适合云原生部署
