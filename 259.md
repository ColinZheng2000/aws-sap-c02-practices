A company is rearchitecting its applications to run on AWS. The company’s infrastructure includes multiple Amazon EC2 instances. The company's development team needs different levels of access. The company wants to implement a policy that requires all Windows EC2 instances to be joined to an Active Directory domain on AWS. The company also wants to implement enhanced security processes such as multi-factor authentication (MFA). The company wants to use managed AWS services wherever possible.

Which solution will meet these requirements?

A. Create an AWS Directory Service for Microsoft Active Directory implementation. Launch an Amazon Workspace. Connect to and use the Workspace for domain security configuration tasks.
B. Create an AWS Directory Service for Microsoft Active Directory implementation. Launch an EC2 instance. Connect to and use the EC2 instance for domain security configuration tasks.
C. Create an AWS Directory Service Simple AD implementation. Launch an EC2 instance. Connect to and use the EC2 instance for domain security configuration tasks.
D. Create an AWS Directory Service Simple AD implementation. Launch an Amazon Workspace. Connect to and use the Workspace for domain security configuration tasks.

# AWS SAP-C02 Analysis — Directory Services for Windows EC2 Domain Join and MFA

## Understanding the Problem

The scenario requires:

- Windows Amazon EC2 instances must join an Active Directory domain
- Support for enterprise security controls such as MFA
- Different levels of access for development teams
- Preference for managed AWS services
- Administrative environment for domain configuration tasks

Key architectural considerations:

- AWS managed Microsoft Active Directory support
- Enterprise-grade AD features
- Integration with AWS services
- Operational overhead minimization
- Secure administrative access pattern

The most important clue is the requirement for:
- Windows domain join
- Enhanced security processes including MFA
- Managed services whenever possible

This strongly points toward:
- AWS Directory Service for Microsoft Active Directory (AWS Managed Microsoft AD)

Simple AD is generally not suitable for enterprise production workloads requiring advanced Microsoft AD features.

- --

# Correct Option

## A. AWS Directory Service for Microsoft Active Directory + Amazon WorkSpaces

### Why this is correct

This option satisfies all requirements while maximizing managed service adoption.

### 1. AWS Managed Microsoft AD supports enterprise Active Directory capabilities

AWS Directory Service for Microsoft Active Directory provides:

- Fully managed Microsoft Active Directory
- Native AD features
- Group Policy support
- Domain join for EC2 Windows instances
- Trust relationships
- Kerberos authentication
- LDAP support
- MFA integrations

This is the correct enterprise solution for production Windows workloads.

Official reference:
- AWS Directory Service Administration Guide
- https://docs.aws.amazon.com/directoryservice/latest/admin-guide/directory_microsoft_ad.html

- --

### 2. Amazon WorkSpaces reduces administrative overhead

Using Amazon WorkSpaces as the administrative workstation aligns with:
- Managed service preference
- Reduced EC2 management overhead
- Persistent secure admin environment
- Easier operational governance

Administrators can:
- Perform AD administration
- Configure Group Policy
- Manage domain join operations
- Handle security configuration tasks

This is preferable to maintaining dedicated administrative EC2 bastion hosts.

- --

### 3. MFA support requirement

AWS Managed Microsoft AD integrates more effectively with enterprise MFA patterns and security tooling than Simple AD.

Examples:
- RADIUS integration
- IAM Identity Center integrations
- Enterprise authentication patterns

Simple AD lacks many advanced enterprise security capabilities.

- --

### 4. Best alignment with “managed services wherever possible”

Managed components used:
- AWS Managed Microsoft AD
- Amazon WorkSpaces

This minimizes:
- OS patching
- Infrastructure management
- AD replication maintenance
- Administrative workstation lifecycle management

- --

# Incorrect Options

## B. AWS Managed Microsoft AD + EC2 administration host

### Why this is incorrect

Although AWS Managed Microsoft AD is correct, launching an EC2 instance for administration introduces unnecessary operational overhead.

Problems:
- Need to manage EC2 lifecycle
- OS patching responsibility
- Security hardening responsibility
- Increased maintenance burden

The requirement explicitly prefers managed AWS services wherever possible.

Amazon WorkSpaces is more aligned with the requirement than EC2.

- --

## C. Simple AD + EC2 administration host

### Why this is incorrect

This option fails primarily because Simple AD is not appropriate for enterprise-grade Microsoft Active Directory requirements.

### Limitations of Simple AD

Simple AD:
- Is Samba-based
- Does not provide full Microsoft AD compatibility
- Lacks many advanced enterprise features
- Has limited MFA/security integration capability
- Is not recommended for complex enterprise Windows environments

AWS documentation specifically positions AWS Managed Microsoft AD for enterprise workloads.

Additionally:
- EC2 administration host increases operational burden

This option violates both:
- Enterprise AD requirement
- Managed service preference

Official reference:
- https://docs.aws.amazon.com/directoryservice/latest/admin-guide/directory_simple_ad.html

- --

## D. Simple AD + Amazon WorkSpaces

### Why this is incorrect

Although WorkSpaces helps satisfy the managed service preference, the core problem remains:

Simple AD is insufficient for enterprise-grade Active Directory requirements.

The major issue is the directory choice, not the admin workstation.

This option may work for small/simple environments but not for enterprise security and MFA-focused architectures.

- --

# Hands-on Knowledge for SAP-C02

## Real Enterprise Pattern

In enterprise AWS environments:

Typical architecture:
- AWS Managed Microsoft AD deployed across multiple AZs
- Windows EC2 instances domain-joined automatically
- AWS Systems Manager used for automation
- Amazon WorkSpaces used for admin access
- MFA enforced through federated identity providers

- --

## Common Production Enhancements

### 1. Automatic Domain Join

Use:
- AWS Systems Manager Automation
- EC2 Launch Templates
- CloudFormation

To automatically:
- Join EC2 instances to domain
- Apply baseline GPOs
- Configure security agents

Official reference:
- AWS Systems Manager Domain Join

- --

### 2. Secure Administrative Access

Modern enterprises avoid:
- Direct RDP exposure
- Public bastion hosts

Instead use:
- Amazon WorkSpaces
- AWS Systems Manager Session Manager
- Zero-trust access models

- --

### 3. Multi-Account AD Design

Large organizations often:
- Share AWS Managed Microsoft AD across accounts
- Use AWS RAM
- Centralize identity services

This is common in:
- Pharmaceutical companies
- Manufacturing environments
- Regulated industries

- --

### 4. Compliance Considerations

Medical and industrial companies often require:
- Central authentication
- MFA enforcement
- Audit logging
- Least privilege administration
- Group Policy governance

AWS Managed Microsoft AD significantly simplifies compliance alignment.
