A company wants to deploy an AWS WAF solution to manage AWS WAF rules across multiple AWS accounts. The accounts are managed under different OUs in AWS Organizations.  
  
Administrators must be able to add or remove accounts or OUs from managed AWS WAF rule sets as needed. Administrators also must have the ability to automatically update and remediate noncompliant AWS WAF rules in all accounts.  
  
Which solution meets these requirements with the LEAST amount of operational overhead?

- A. Use AWS Firewall Manager to manage AWS WAF rules across accounts in the organization. Use an AWS Systems Manager Parameter Store parameter to store account numbers and OUs to manage. Update the parameter as needed to add or remove accounts or OUs. Use an Amazon EventBridge rule to identify any changes to the parameter and to invoke an AWS Lambda function to update the security policy in the Firewall Manager administrative account.
- B. Deploy an organization-wide AWS Config rule that requires all resources in the selected OUs to associate the AWS WAF rules. Deploy automated remediation actions by using AWS Lambda to fix noncompliant resources. Deploy AWS WAF rules by using an AWS CloudFormation stack set to target the same OUs where the AWS Config rule is applied.
- C. Create AWS WAF rules in the management account of the organization. Use AWS Lambda environment variables to store account numbers and OUs to manage. Update environment variables as needed to add or remove accounts or OUs. Create cross-account IAM roles in member accounts. Assume the roles by using AWS Security Token Service (AWS STS) in the Lambda function to create and update AWS WAF rules in the member accounts.
- D. Use AWS Control Tower to manage AWS WAF rules across accounts in the organization. Use AWS Key Management Service (AWS KMS) to store account numbers and OUs to manage. Update AWS KMS as needed to add or remove accounts or OUs. Create IAM users in member accounts. Allow AWS Control Tower in the management account to use the access key and secret access key to create and update AWS WAF rules in the member accounts.

# AWS WAF Multi-Account Management Solution Analysis

## Understanding the Problem
The company needs to centrally manage AWS WAF rules across multiple AWS accounts in an AWS Organizations structure. Key requirements include:
- Cross-account WAF rule management
- Dynamic addition/removal of accounts/OUs from rule sets
- Automatic remediation of noncompliant rules
- Minimal operational overhead

## Services Explanation

**AWS Firewall Manager**: Fully managed service for configuring and managing AWS WAF rules across accounts in AWS Organizations. Provides centralized security policy management with automatic compliance enforcement.

**AWS Systems Manager Parameter Store**: Secure storage for configuration data management. Allows hierarchical parameter organization with encryption capabilities.

**Amazon EventBridge**: Serverless event bus service that routes events from AWS services to targets like Lambda functions.

**AWS Lambda**: Serverless compute service that runs code in response to events without provisioning servers.

**AWS Organizations**: Policy-based management for multiple AWS accounts, including organizational units (OUs) for logical grouping.

## Correct Option Explanation: A

**Why Option A Meets Requirements:**
1. **AWS Firewall Manager** natively supports cross-account WAF rule management through AWS Organizations
2. Automatic policy application to new accounts in specified OUs
3. Built-in compliance monitoring and automatic remediation
4. Parameter Store provides dynamic account/OU list management
5. EventBridge-Lambda automation updates policies when parameters change
6. Minimal operational overhead through managed services

**Official AWS Documentation Support:**
- [AWS Firewall Manager Documentation](https://docs.aws.amazon.com/waf/latest/developerguide/fms-chapter.html) confirms centralized WAF rule management across organizations
- [Firewall Manager Policy Management](https://docs.aws.amazon.com/fms/2018-01-01/APIReference/API_Policy.html) shows native OU/account targeting capabilities

## Incorrect Options Explanation

### Option B
- **AWS Config** is for compliance auditing, not proactive security management
- **CloudFormation Stack Sets** require manual updates for account/OU changes
- Higher operational overhead due to multiple service coordination
- No native automatic remediation for WAF rule compliance

### Option C
- Custom Lambda solution creates significant operational overhead
- Manual cross-account IAM role management required
- No built-in compliance monitoring or remediation
- Environment variables aren't dynamic for account/OU changes

### Option D
- **AWS Control Tower** is for landing zone setup, not WAF management
- **AWS KMS** is for encryption, not account/OU list management
- IAM user creation violates security best practices
- No native WAF rule management capabilities in Control Tower

## Key Takeaways
- Firewall Manager is the purpose-built service for cross-account WAF management
- Native AWS Organizations integration minimizes custom configuration
- Managed services reduce operational overhead compared to custom solutions
- Parameter Store + EventBridge provides dynamic policy targeting
